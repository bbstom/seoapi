# API 健康检测 - 快速参考

## 状态圆点说明

| 圆点 | 状态 | 说明 | 动画 |
|------|------|------|------|
| 🟢 | 在线 | API 连接正常，可以使用 | 脉冲动画 |
| 🔴 | 离线 | API 无法连接或超时 | 无 |
| 🟡 | 检查中 | 正在检测 API 状态 | 脉冲动画 |
| ⚪ | 已禁用 | 配置被手动禁用 | 无 |
| ⚪ | 未知 | 尚未检测状态 | 无 |

## 检测机制

- **首次检测：** 打开页面立即检测
- **自动刷新：** 每 30 秒检测一次
- **超时时间：** 3 秒
- **检测方式：** 轻量级连接测试

## 快速测试

```bash
# 1. 打开浏览器
http://localhost:3000

# 2. 进入系统配置 → 外部 API 配置

# 3. 观察圆点颜色
🟢 = API 在线
🔴 = API 离线
🟡 = 检查中

# 4. 等待 30 秒，观察自动刷新
```

## 技术要点

**前端：**
- 使用 `useEffect` 实现自动检测
- 每 30 秒调用健康检查接口
- 根据响应更新圆点颜色

**后端：**
- 新增 `/api/api-configs/health-check` 接口
- 3 秒超时的快速连接测试
- 返回在线/离线状态

**性能：**
- 轻量级检测，不获取模型列表
- 只在配置页面激活时检测
- 顺序检测，避免并发压力
