# 数据库初始化 - 快速参考

## 🚀 快速开始（推荐方案）

```bash
# 1. 进入项目目录
cd /www/wwwroot/api.vpno.eu.org/seoapi

# 2. 删除并重建数据库
mysql -h 172.16.254.100 -u root -pUbuntu123! -e "DROP DATABASE IF EXISTS seoapi; CREATE DATABASE seoapi CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"

# 3. 运行新的初始化脚本
node scripts/init-database-from-sql.js

# 4. 验证结果
node scripts/diagnose-database.js

# 5. 重启服务
pm2 restart seoapi
```

## 🔍 诊断命令

```bash
# 检查数据库结构
node scripts/diagnose-database.js

# 查看表列表
mysql -h 172.16.254.100 -u root -pUbuntu123! seoapi -e "SHOW TABLES;"

# 查看 api_configs 表结构
mysql -h 172.16.254.100 -u root -pUbuntu123! seoapi -e "DESCRIBE api_configs;"

# 查看 node_health_logs 表结构
mysql -h 172.16.254.100 -u root -pUbuntu123! seoapi -e "DESCRIBE node_health_logs;"
```

## ✅ 关键字段检查

### api_configs 表必须包含：
- `base_url` (不是 api_url)
- `weight`
- `health_status`
- `last_check_at`
- `avg_latency`

### node_health_logs 表必须包含：
- `connectivity`
- `authentication`
- `quota_status`
- `response_code`
- `response_time`

## 🛠️ 故障排除

### 问题：字段不存在错误

```bash
# 解决方案：使用新脚本重新初始化
node scripts/init-database-from-sql.js
```

### 问题：外键约束错误

```bash
# 解决方案：删除数据库重建
mysql -h 172.16.254.100 -u root -pUbuntu123! -e "DROP DATABASE seoapi; CREATE DATABASE seoapi CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"
node scripts/init-database-from-sql.js
```

### 问题：权限错误

```bash
# 检查数据库用户权限
mysql -h 172.16.254.100 -u root -pUbuntu123! -e "SHOW GRANTS FOR 'root'@'%';"
```
