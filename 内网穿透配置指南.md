# 内网穿透配置指南

## 🌐 问题说明

当你使用内网穿透（如 ngrok、frp、花生壳等）将本地服务映射到公网时，小旋风可能无法访问。

## ✅ 服务器配置检查

### 1. 确认服务监听地址

服务已配置为监听 `0.0.0.0`，这是正确的：

```javascript
app.listen(PORT, '0.0.0.0', () => {
    // ...
});
```

这意味着服务会接受来自所有网络接口的连接。

### 2. 确认服务正在运行

```powershell
# 检查服务是否运行
netstat -ano | findstr :8000

# 应该看到类似输出：
# TCP    0.0.0.0:8000           0.0.0.0:0              LISTENING       12345
```

如果看到 `127.0.0.1:8000` 而不是 `0.0.0.0:8000`，说明配置有问题。

## 🔥 Windows 防火墙配置

### 方法 1：通过 PowerShell 添加规则（推荐）

**以管理员身份运行 PowerShell：**

```powershell
# 添加入站规则
New-NetFirewallRule -DisplayName "SEO API Service" `
    -Direction Inbound `
    -Protocol TCP `
    -LocalPort 8000 `
    -Action Allow `
    -Profile Any

# 验证规则
Get-NetFirewallRule -DisplayName "SEO API Service"
```

### 方法 2：通过 Windows 防火墙界面

1. 打开"Windows Defender 防火墙"
2. 点击"高级设置"
3. 点击"入站规则"
4. 点击"新建规则"
5. 选择"端口" → 下一步
6. 选择"TCP"，输入端口 `8000` → 下一步
7. 选择"允许连接" → 下一步
8. 全选（域、专用、公用）→ 下一步
9. 输入名称"SEO API Service" → 完成

### 方法 3：临时关闭防火墙测试

**⚠️ 仅用于测试，不推荐长期使用**

```powershell
# 以管理员身份运行
Set-NetFirewallProfile -Profile Domain,Public,Private -Enabled False

# 测试完成后记得重新启用
Set-NetFirewallProfile -Profile Domain,Public,Private -Enabled True
```

## 🌉 内网穿透工具配置

### ngrok

**安装和使用：**

```powershell
# 下载 ngrok: https://ngrok.com/download

# 启动内网穿透
ngrok http 8000

# 会显示公网地址，例如：
# Forwarding  https://abc123.ngrok.io -> http://localhost:8000
```

**小旋风配置：**

```
API地址: https://abc123.ngrok.io/api/rewrite
post格式: txt={word}&mode=seo_fast&sim=1
Authorization: your_api_key
成功标志: errcode=0
返回错误的字段: errmsg
伪原创后的内容字段: data
```

**注意事项：**
- ✅ ngrok 免费版每次启动 URL 会变化
- ✅ 支持 HTTPS
- ✅ 有请求日志

### frp（推荐用于长期使用）

**客户端配置 `frpc.ini`：**

```ini
[common]
server_addr = your-frp-server.com
server_port = 7000
token = your_token

[seoapi]
type = http
local_ip = 127.0.0.1
local_port = 8000
custom_domains = seoapi.yourdomain.com
```

**启动：**

```powershell
frpc.exe -c frpc.ini
```

### 花生壳

1. 下载并安装花生壳客户端
2. 登录账号
3. 添加内网映射
   - 内网主机：127.0.0.1
   - 内网端口：8000
   - 外网端口：自动分配或指定
4. 获取外网访问地址

### Cloudflare Tunnel（推荐）

**安装 cloudflared：**

```powershell
# 下载: https://github.com/cloudflare/cloudflared/releases

# 登录
cloudflared tunnel login

# 创建隧道
cloudflared tunnel create seoapi

# 配置隧道
# 创建 config.yml
```

**config.yml：**

```yaml
tunnel: <tunnel-id>
credentials-file: C:\Users\YourName\.cloudflared\<tunnel-id>.json

ingress:
  - hostname: seoapi.yourdomain.com
    service: http://localhost:8000
  - service: http_status:404
```

**启动：**

```powershell
cloudflared tunnel run seoapi
```

## 🐛 常见问题排查

### 问题 1：连接超时

**症状：** 小旋风提示"连接超时"或"无法连接"

**可能原因：**
1. Windows 防火墙阻止
2. 服务未启动
3. 端口映射错误

**排查步骤：**

```powershell
# 1. 检查服务是否运行
netstat -ano | findstr :8000

# 2. 检查防火墙规则
Get-NetFirewallRule -DisplayName "SEO API Service"

# 3. 测试本地访问
curl http://localhost:8000/health

# 4. 测试局域网访问
curl http://192.168.1.100:8000/health
```

### 问题 2：返回空内容

**症状：** 小旋风提示"返回空内容"

**可能原因：**
1. 请求到达服务器但响应被拦截
2. 内网穿透工具配置问题
3. HTTPS/HTTP 混用

**排查步骤：**

1. **查看服务器日志**
   - 如果有请求日志，说明请求到达了
   - 查看响应是否成功发送

2. **检查内网穿透日志**
   - ngrok: 查看 ngrok 控制台
   - frp: 查看 frpc 日志

3. **测试直接访问**
   ```powershell
   curl https://your-tunnel-url/health
   ```

### 问题 3：HTTPS 证书问题

**症状：** 小旋风提示"证书错误"

**解决方案：**

1. **使用 ngrok**（自动提供 HTTPS）
2. **使用 Cloudflare Tunnel**（自动提供 HTTPS）
3. **配置反向代理**（Nginx + Let's Encrypt）

### 问题 4：速率限制

**症状：** 请求被限流

**原因：** 内网穿透工具可能有速率限制

**解决方案：**
1. 升级到付费版
2. 使用自建 frp 服务器
3. 使用 Cloudflare Tunnel（无限制）

## 🎯 推荐方案

### 开发测试（短期）

**使用 ngrok：**

```powershell
# 1. 启动服务
node server.js

# 2. 启动 ngrok（另一个窗口）
ngrok http 8000

# 3. 使用 ngrok 提供的 URL
# 例如: https://abc123.ngrok.io
```

**优点：**
- ✅ 简单快速
- ✅ 自动 HTTPS
- ✅ 有请求日志

**缺点：**
- ❌ 免费版 URL 会变
- ❌ 有速率限制

### 生产环境（长期）

**使用 Cloudflare Tunnel：**

```powershell
# 1. 安装 cloudflared
# 2. 配置隧道
# 3. 绑定域名
# 4. 启动隧道
```

**优点：**
- ✅ 免费
- ✅ 稳定
- ✅ 自动 HTTPS
- ✅ 无速率限制
- ✅ 固定域名

**缺点：**
- ❌ 需要域名
- ❌ 配置稍复杂

### 自建服务器

**使用 frp + VPS：**

1. 在 VPS 上部署 frps（服务端）
2. 在本地运行 frpc（客户端）
3. 配置域名和 Nginx

**优点：**
- ✅ 完全控制
- ✅ 无限制
- ✅ 稳定

**缺点：**
- ❌ 需要 VPS
- ❌ 需要配置

## 🔧 快速测试脚本

**创建 `test-public.ps1`：**

```powershell
# 测试公网访问
param(
    [string]$url = "https://your-tunnel-url.com"
)

Write-Host "测试公网访问: $url" -ForegroundColor Cyan
Write-Host ""

# 测试健康检查
Write-Host "1. 测试健康检查..." -ForegroundColor Yellow
try {
    $response = Invoke-WebRequest -Uri "$url/health" -UseBasicParsing
    Write-Host "✓ 健康检查成功" -ForegroundColor Green
    Write-Host $response.Content
} catch {
    Write-Host "✗ 健康检查失败" -ForegroundColor Red
    Write-Host $_.Exception.Message
}

Write-Host ""

# 测试 API
Write-Host "2. 测试 API 接口..." -ForegroundColor Yellow
$apiKey = "sk_f7747efeefece9fbeb79bfecd825e01ff427c2f343acd398f6f3713469906d00"
$headers = @{
    "Authorization" = $apiKey
    "Content-Type" = "application/x-www-form-urlencoded"
}
$body = "txt=测试文本&mode=seo_fast&sim=1"

try {
    $response = Invoke-WebRequest -Uri "$url/api/rewrite" `
        -Method POST `
        -Headers $headers `
        -Body $body `
        -UseBasicParsing
    
    Write-Host "✓ API 测试成功" -ForegroundColor Green
    $json = $response.Content | ConvertFrom-Json
    Write-Host "errcode: $($json.errcode)"
    Write-Host "data: $($json.data.Substring(0, [Math]::Min(50, $json.data.Length)))..."
} catch {
    Write-Host "✗ API 测试失败" -ForegroundColor Red
    Write-Host $_.Exception.Message
}
```

**使用：**

```powershell
.\test-public.ps1 -url "https://abc123.ngrok.io"
```
