# 数据库字段整合说明

## 背景

在开发过程中，添加了两个新功能需要的数据库字段：
1. 令牌负载均衡模型多选 - 需要 `allowed_models` 字段
2. 数据看板显示API节点 - 需要 `node_id` 和 `node_name` 字段

这些字段已经整合到初始化脚本中，新部署环境无需额外迁移。

## 已整合的字段

### 1. api_keys 表
**新增字段：** `allowed_models`
- 类型：JSON
- 说明：允许使用的模型列表（支持多选）
- 位置：在 `default_model` 字段之前
- 用途：令牌负载均衡时限制可用模型

### 2. api_logs 表
**新增字段：** `node_id` 和 `node_name`
- `node_id`：INT，API节点ID
- `node_name`：VARCHAR(100)，API节点名称
- 位置：在 `base_url` 字段之后
- 索引：添加了 `idx_node_id` 索引
- 用途：数据看板显示调用的节点信息

## 初始化脚本更新

**文件：** `scripts/init-database-from-sql.js`

### api_keys 表结构（部分）
```sql
CREATE TABLE IF NOT EXISTS api_keys (
  ...
  load_balance_nodes JSON NULL COMMENT '负载均衡节点池（节点ID数组）',
  load_balance_strategy VARCHAR(50) DEFAULT 'round_robin',
  allowed_models JSON NULL COMMENT '允许使用的模型列表（JSON数组）',  -- 新增
  default_model VARCHAR(255) NULL DEFAULT NULL,
  ...
)
```

### api_logs 表结构（部分）
```sql
CREATE TABLE IF NOT EXISTS api_logs (
  ...
  base_url VARCHAR(255) NULL DEFAULT NULL,
  node_id INT NULL COMMENT 'API节点ID',          -- 新增
  node_name VARCHAR(100) NULL COMMENT 'API节点名称',  -- 新增
  api_type VARCHAR(50) NULL DEFAULT NULL,
  ...
  INDEX idx_node_id (node_id)  -- 新增索引
)
```

## 迁移脚本（仅用于已有数据库）

如果数据库已存在但缺少这些字段，使用以下迁移脚本：

### 1. 添加 allowed_models 字段
```bash
node scripts/add-allowed-models.js
```

**作用：**
- 添加 `allowed_models` 字段到 `api_keys` 表
- 迁移现有的 `default_model` 数据到 `allowed_models`

### 2. 添加节点信息字段
```bash
node scripts/add-node-info-to-logs.js
```

**作用：**
- 添加 `node_id` 字段到 `api_logs` 表
- 添加 `node_name` 字段到 `api_logs` 表
- 添加 `idx_node_id` 索引

## 部署场景

### 场景 1：全新部署
**操作：** 直接使用初始化脚本
```bash
node scripts/init-database-from-sql.js
```
**结果：** 所有字段都已包含，无需额外迁移

### 场景 2：已有数据库（缺少新字段）
**操作：** 运行迁移脚本
```bash
# 添加 allowed_models 字段
node scripts/add-allowed-models.js

# 添加节点信息字段
node scripts/add-node-info-to-logs.js

# 重启服务
pm2 restart seoapi
```

### 场景 3：已有数据库（需要重建）
**操作：** 备份数据后重新初始化
```bash
# 1. 备份数据
mysqldump -h 172.16.254.100 -u root -pUbuntu123! seoapi > backup.sql

# 2. 重建数据库
mysql -h 172.16.254.100 -u root -pUbuntu123! -e "DROP DATABASE seoapi; CREATE DATABASE seoapi CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"

# 3. 初始化
node scripts/init-database-from-sql.js

# 4. 恢复数据（如需要）
mysql -h 172.16.254.100 -u root -pUbuntu123! seoapi < backup.sql
```
