# 小旋风故障排查指南

## 问题：请求API失败，返回空内容

当小旋风显示"请求API失败，返回空内容"时，按以下步骤排查：

---

## 第一步：确认服务正在运行

### Windows 系统

```bash
# 方法 1: 检查端口占用
netstat -ano | findstr :8000

# 方法 2: 访问健康检查接口
curl http://localhost:8000/health
```

### 启动服务

```bash
# 进入项目目录
cd seoapi-nodejs

# 启动服务
node server.js
```

**预期输出：**
```
========================================
  SEO API - Claude AI 文本改写服务
========================================

服务地址: http://localhost:8000
API 文档: http://localhost:8000/api/docs
Web 界面: http://localhost:8000

按 Ctrl+C 停止服务
========================================

API Key 状态: ✓ 已配置
```

---

## 第二步：验证 API 配置

### 1. 检查用户 API Key

打开 `seoapi-nodejs/data/users.json`，确认：

```json
{
  "admin": {
    "apiKey": "sk_xxxxx...",  // 你的 API Key
    "claudeApiKey": "sk-xxxxx...",  // Claude API Key
    "claudeBaseURL": "https://api123.icu",  // API 地址
    "defaultModel": "claude-sonnet-4-5-20250929",
    "defaultMode": "seo_fast"
  }
}
```

### 2. 测试 API 连接

**使用 PowerShell 测试：**

```powershell
# 替换 YOUR_API_KEY 为你的实际 API Key
$apiKey = "sk_f7747efeefece9fbeb79bfecd825e01ff427c2f343acd398f6f3713469906d00"
$headers = @{
    "Authorization" = $apiKey
    "Content-Type" = "application/x-www-form-urlencoded"
}
$body = "txt=人工智能技术正在快速发展&mode=seo_fast&sim=1"

Invoke-WebRequest -Uri "http://localhost:8000/api/rewrite" `
    -Method POST `
    -Headers $headers `
    -Body $body `
    -UseBasicParsing
```

**预期响应：**
```json
{
  "errcode": "0",
  "errmsg": "",
  "data": "改写后的文本...",
  "like": "0.3521",
  "_meta": {
    "mode": "seo_fast",
    "model": "claude-sonnet-4-5-20250929",
    "duration": 2.5
  }
}
```

## 第三步：检查小旋风配置

### 正确的配置格式

```
API名称: Claude AI 伪原创
API地址: http://localhost:8000/api/rewrite
post格式: txt={word}&mode=seo_fast&sim=1
Authorization: sk_f7747efeefece9fbeb79bfecd825e01ff427c2f343acd398f6f3713469906d00
成功标志: errcode=0
返回错误的字段: errmsg
伪原创后的内容字段: data
```

### 常见配置错误

❌ **错误 1：API 地址多了斜杠**
```
错误: http://localhost:8000//api/rewrite
正确: http://localhost:8000/api/rewrite
```

❌ **错误 2：Authorization 字段名错误**
```
错误: apikey: sk_xxxxx
正确: Authorization: sk_xxxxx
```

❌ **错误 3：post 格式使用了错误的变量**
```
错误: txt=[content]&mode=seo_fast
正确: txt={word}&mode=seo_fast&sim=1
```

❌ **错误 4：成功标志格式错误**
```
错误: errcode="0"
正确: errcode=0
```

**错误 1：API Key 无效**

```
**解决方案：** 检查 `claudeApiKey` 是否正确

**错误 2：模型不可用**
```
[admin] 改写失败: model_not_found

**解决方案：** 使用支持的模型（见下方列表）

**错误 3：超时**
```

```
**解决方案：** 
- 使用更快的模型（Haiku 或 Sonnet）
- 减少文本长度
- 检查网络连接

## 第六步：常见错误代码

| 错误码 | 错误信息 | 原因 | 解决方案 |
|--------|---------|------|---------|
| 100202 | 请求缺少 apikey | 未提供 API Key | 检查 Authorization 字段 |
| 100203 | 无效的 apikey | API Key 错误 | 检查 users.json 中的 apiKey |
| 100203 | 用户未配置 Claude API Key | 未配置 Claude Key | 在 Web 界面配置 Claude API Key |
| 100301 | 所选模型不可用 | 模型不存在 | 使用支持的模型 |
| 100301 | 无法连接到 API 服务器 | 网络问题 | 检查 claudeBaseURL 配置 |
| 200201 | 传进参数为空 | 文本为空 | 检查 txt 参数 |
| 200500 | 内容长度超限 | 文本太长 | 限制在 5000 字符以内 |

## 第九步：超时问题解决

如果遇到 504 超时错误：

### 原因分析

1. **文本太长** - 超过 3000 字符
2. **模型太慢** - 使用了 Opus 模型
3. **网络问题** - API 中转服务响应慢

### 解决方案

**方案 1：使用更快的模型**

```
# 推荐配置（速度优先）
post格式: txt={word}&mode=seo_fast&model=claude-haiku-4-5-20251001&sim=1

# 推荐配置（平衡）
post格式: txt={word}&mode=seo_fast&model=claude-sonnet-4-5-20250929&sim=1
```

**方案 2：限制文本长度**

在小旋风中设置：
- 最大采集长度：3000 字符
- 超长文本：分段处理

**方案 3：根据文本长度选择模型**

| 文本长度 | 推荐模型 | 预计耗时 |
|---------|---------|---------|
| < 500 字符 | Opus 4.6 | 3-5 秒 |
| 500-1500 字符 | Sonnet 4.5 | 2-4 秒 |
| 1500-3000 字符 | Sonnet 4.5 | 4-8 秒 |
| > 3000 字符 | Haiku 4.5 | 3-6 秒 |

## 第十步：Web 界面配置

### 访问 Web 界面

1. 打开浏览器访问：`http://localhost:8000`
2. 使用默认账号登录：
   - 用户名：`admin`
   - 密码：`admin123`

### 配置 Claude API

1. 登录后点击"配置"
2. 填写 Claude API Key
3. 设置 API 地址（默认：`https://api123.icu`）
4. 选择默认模型和模式
5. 保存配置

### 测试改写功能

1. 在 Web 界面输入测试文本
2. 选择改写模式
3. 点击"改写"按钮
4. 查看结果

## 总结

"请求API失败，返回空内容"通常是以下原因之一：

1. ❌ 服务未启动
2. ❌ API Key 配置错误
3. ❌ 小旋风配置格式错误
4. ❌ 网络连接问题
5. ❌ Claude API Key 未配置或无效

按照本指南逐步排查，99% 的问题都能解决！🚀
