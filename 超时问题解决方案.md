# 504 Gateway Timeout 超时问题解决方案

## 🔍 问题分析

### 错误信息
```
504 Gateway Time-out
nginx/1.26.3
```

### 发生场景
- **模式**: seo_fast
- **模型**: claude-opus-4-6
- **文本长度**: 4812 字符
- **错误原因**: 处理时间超过 60 秒（nginx 超时限制）

## 📊 根本原因

### 1. 模型处理速度差异

| 模型 | 速度 | 适用文本长度 | 说明 |
|------|------|--------------|------|
| **Haiku 4.5** | ⚡⚡⚡⚡⚡ | 0-10000 字 | 最快，适合长文本 |
| **Sonnet 4.5** | ⚡⚡⚡⚡ | 0-5000 字 | 平衡，推荐使用 |
| **Sonnet 4.6** | ⚡⚡⚡ | 0-3000 字 | 较慢 |
| **Opus 4.5** | ⚡⚡ | 0-2000 字 | 慢，高质量 |
| **Opus 4.6** | ⚡ | 0-1000 字 | 最慢，仅短文本 |

### 2. 超时限制

- **api123.icu nginx 超时**: 60 秒
- **Opus 4.6 处理 4812 字**: 需要 70-90 秒
- **结果**: 504 Gateway Timeout

### 3. 文本长度影响

处理时间 ≈ 文本长度 × 模型速度系数

**Opus 4.6 处理时间估算**:
- 500 字: ~10 秒 ✅
- 1000 字: ~20 秒 ✅
- 2000 字: ~40 秒 ✅
- 3000 字: ~60 秒 ⚠️ 临界
- 4000 字: ~80 秒 ❌ 超时
- 5000 字: ~100 秒 ❌ 超时

## ✅ 解决方案

### 方案 1：根据文本长度选择模型（推荐）⭐

#### 配置规则

| 文本长度 | 推荐模型 | 原因 |
|----------|----------|------|
| < 500 字 | Opus 4.6 | 可以使用最高质量 |
| 500-2000 字 | Sonnet 4.5 | 平衡质量和速度 |
| 2000-5000 字 | Sonnet 4.5 + seo_fast | 快速模式 |
| > 5000 字 | Haiku 4.5 + seo_fast | 最快速度 |

#### 小旋风配置示例

**短文本（< 500 字）**:
```
post 格式: txt={word}&mode=seo_original&model=claude-opus-4-6&sim=1
```

**中等文本（500-2000 字）**:
```
post 格式: txt={word}&mode=seo_spider&model=claude-sonnet-4-5-20250929&sim=1
```

**长文本（2000-5000 字）**:
```
post 格式: txt={word}&mode=seo_fast&model=claude-sonnet-4-5-20250929&sim=1
```

**超长文本（> 5000 字）**:
```
post 格式: txt={word}&mode=seo_fast&model=claude-haiku-4-5-20251001&sim=1
```

## 💡 Web 界面默认配置

### 设置步骤

1. 登录 Web 界面: `http://localhost:8000`
2. 点击右上角用户名
3. 找到"默认配置"部分
4. 设置:
   - **默认模型**: `claude-sonnet-4-5-20250929`
   - **默认模式**: `seo_spider`
5. 点击"保存默认配置"

### 效果

小旋风调用时，如果不指定 `model` 参数，将自动使用 Sonnet 4.5，避免超时。

```
post 格式: txt={word}&mode=seo_spider&sim=1
```

等同于:

```
post 格式: txt={word}&mode=seo_spider&model=claude-sonnet-4-5-20250929&sim=1
```

4. ✅ **分段处理**: 超长文本分段改写
