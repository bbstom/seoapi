# 令牌负载均衡增强 - 实现方案

## 需求
1. 负载均衡模式下支持多选API节点
2. 支持选择负载均衡策略（轮询、加权、最少连接等）
3. 支持指定默认模型

## 数据库变更

已完成数据库升级，新增字段：

```sql
ALTER TABLE api_keys ADD COLUMN load_balance_nodes JSON DEFAULT NULL;
ALTER TABLE api_keys ADD COLUMN load_balance_strategy VARCHAR(50) DEFAULT 'round_robin';
ALTER TABLE api_keys ADD COLUMN default_model VARCHAR(255) DEFAULT NULL;
```

### 字段说明
- `load_balance_nodes`: JSON数组，存储参与负载均衡的节点ID列表，如 `[1, 2, 3]`
- `load_balance_strategy`: 负载均衡策略
  - `round_robin`: 轮询（默认）
  - `weighted`: 加权轮询
  - `least_connections`: 最少连接
  - `random`: 随机
- `default_model`: 默认使用的模型名称

### 令牌管理器更新

```javascript
// lib/tokenManager.js
async updateTokenNodeConfig(username, tokenId, config) {
  const {
    nodeStrategy,
    loadBalanceStrategy,
    loadBalanceNodes,
    defaultModel,
    fixedNodeId,
    fixedModel
  } = config;
  
  await pool.query(`
    UPDATE api_keys 
    SET node_strategy = ?,
        load_balance_strategy = ?,
        load_balance_nodes = ?,
        default_model = ?,
        fixed_node_id = ?,
        fixed_model = ?
    WHERE id = ? AND user_id = ?
  `, [
    nodeStrategy,
    loadBalanceStrategy || 'round_robin',
    loadBalanceNodes ? JSON.stringify(loadBalanceNodes) : null,
    defaultModel,
    fixedNodeId,
    fixedModel,
    tokenId,
    userId
  ]);
}
```

## 负载均衡策略实现

### 1. 轮询 (Round Robin)
按顺序依次选择节点

### 2. 加权轮询 (Weighted Round Robin)
根据节点权重分配请求

### 3. 最少连接 (Least Connections)
选择当前连接数最少的节点

### 4. 随机 (Random)
随机选择一个节点

## 使用场景

### 场景1: 限制节点范围
用户有5个节点，但某个令牌只想在节点A和B之间负载均衡

### 场景2: 不同策略
- 高优先级令牌使用"最少连接"策略
- 普通令牌使用"轮询"策略

### 场景3: 固定模型
负载均衡模式下，所有节点都使用 claude-sonnet-4 模型
