## 说明

本系统支持两种 API 格式：
1. **Anthropic 官方格式** - 使用 Anthropic SDK
2. **OpenAI 兼容格式** - 使用 OpenAI API 格式（如 fucaixie.xyz）

两种格式的模型名称可能不同，但系统会自动处理。

## 使用说明

### 配置 Anthropic 官方 API

```
Claude Base URL: https://api.anthropic.com
Claude API Key: sk-ant-api03-xxxxx
默认模型: claude-sonnet-4-5-20250929
```

### 配置 OpenAI 兼容 API（fucaixie.xyz）

```
Claude Base URL: https://fucaixie.xyz/v1
Claude API Key: sk-xxxxx
默认模型: claude-opus-4-6
```


## 注意事项

### 1. 模型可用性

不同的 API 服务商可能支持不同的模型：
- Anthropic 官方：支持所有模型
- fucaixie.xyz：支持部分模型（需要在后台配置）
- api123.icu：支持部分模型

如果遇到 `model_not_found` 错误，说明当前 API 服务商不支持该模型，请：
1. 切换到其他模型
2. 或联系服务商开通该模型

### 2. 模型名称

系统会自动处理模型名称，你只需要：
1. 在 Web 界面选择模型
2. 或在 API 请求中指定模型 ID

系统会根据配置的 Base URL 自动使用正确的格式。

### 3. 默认模型

建议在 Web 界面设置默认模型：
- 开发环境：claude-haiku-4-5-20251001（快速测试）
- 生产环境：claude-sonnet-4-5-20250929（平衡性能）
- 高质量场景：claude-opus-4-6（最佳效果）

## 测试模型

### 方法 1：Web 界面测试

1. 访问 `http://localhost:8000`
2. 登录后进入设置
3. 选择模型
4. 点击"测试改写"

### 方法 2：API 测试

```bash
curl -X POST http://localhost:8000/api/rewrite \
  -H "Authorization: your_api_key" \
  -H "Content-Type: application/json" \
  -d '{
    "txt": "测试文本",
    "mode": "seo_fast",
    "model": "claude-opus-4-6"
  }'
```

### 方法 3：小旋风测试

在小旋风的 post 格式中指定模型：
```
txt={word}&mode=seo_fast&model=claude-opus-4-6
```

## 常见问题

### Q1: 为什么我选择的模型不可用？

A: 可能原因：
1. API 服务商不支持该模型
2. 模型名称拼写错误
3. API Key 没有该模型的权限

解决方法：
1. 查看服务器日志确认错误信息
2. 切换到其他模型
3. 联系 API 服务商

### Q2: 不同模型的效果有什么区别？

A: 主要区别：
- Haiku：速度快，适合简单改写
- Sonnet：平衡性能，适合大多数场景
- Opus：质量高，适合复杂改写

### Q3: 如何选择最合适的模型？

A: 建议：
1. 先用 Sonnet 测试
2. 如果速度慢，换 Haiku
3. 如果质量不够，换 Opus

### Q4: 模型名称在两个网站不一样怎么办？

A: 系统已经自动处理，你只需要：
1. 使用系统中的模型 ID
2. 系统会根据 Base URL 自动转换
