# 数据库初始化指南

## 问题背景

生产环境多次初始化数据库后，表结构与代码不匹配，导致以下错误：

1. `api_configs` 表缺少字段：`weight`, `base_url`, `priority`, `description`, `avg_latency` 等
2. `node_health_logs` 表缺少字段：`connectivity`, `authentication`, `quota_status` 等
3. 使用了错误的字段名：`api_url` 而不是 `base_url`

## 解决方案

基于生产环境的完整数据库备份 `seoapi2.sql` 创建了新的初始化脚本。

## 使用步骤

### 方案 1：完全重建数据库（推荐）

如果数据库中没有重要数据，建议完全重建：

```bash
# 1. 备份当前数据库（如果有重要数据）
mysqldump -h 172.16.254.100 -u root -p seoapi > backup_$(date +%Y%m%d_%H%M%S).sql

# 2. 删除并重建数据库
mysql -h 172.16.254.100 -u root -p -e "DROP DATABASE IF EXISTS seoapi; CREATE DATABASE seoapi CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"

# 3. 运行新的初始化脚本
cd /www/wwwroot/api.vpno.eu.org/seoapi
node scripts/init-database-from-sql.js
```

### 方案 2：诊断并修复

如果想保留现有数据，先诊断问题：

```bash
# 1. 运行诊断脚本
node scripts/diagnose-database.js

# 2. 根据诊断结果，手动添加缺失字段
# 或者导入 seoapi2.sql 文件
mysql -h 172.16.254.100 -u root -p seoapi < seoapi2.sql
```

## 脚本说明

### 1. init-database-from-sql.js

完整的数据库初始化脚本，基于 `seoapi2.sql` 的实际结构创建。

**特点：**
- 包含所有必需字段
- 字段名称、类型、默认值与生产环境一致
- 自动创建默认管理员账号
- 包含所有索引和外键约束

**关键表结构：**

#### api_configs 表
```sql
- base_url (不是 api_url)
- weight (负载均衡权重)
- health_status (健康状态)
- last_check_at (最后检查时间)
- avg_latency (平均延迟)
- active_connections (活跃连接数)
- total_connections (总连接数)
```

#### node_health_logs 表
```sql
- connectivity (连接状态)
- authentication (认证状态)
- auth_error_code (认证错误码)
- quota_status (额度状态)
- quota_total (总额度)
- quota_used (已用额度)
- quota_remaining (剩余额度)
- quota_percentage (剩余额度百分比)
- response_code (HTTP状态码)
- response_time (响应时间)
```

### 2. diagnose-database.js

数据库诊断脚本，检查当前数据库结构。

**功能：**
- 检查所有关键表是否存在
- 对比实际字段与期望字段
- 列出缺失字段和多余字段
- 显示每个表的记录数

**使用场景：**
- 初始化前检查数据库状态
- 初始化后验证结果
- 排查字段缺失问题

```bash

# 2. 检查表结构

mysql -h 172.16.254.100 -u root -p seoapi -e "SHOW COLUMNS FROM node_health_logs;"
```

## 常见问题

### Q1: 初始化后仍然报错字段不存在

**原因：** 可能使用了旧的初始化脚本

**解决：** 确保使用 `init-database-from-sql.js` 而不是 `init-database.js`

### Q2: 外键约束错误

**原因：** 表创建顺序不正确

**解决：** 脚本已按正确顺序创建表（先创建父表，再创建子表）

### Q3: 字符集问题

**原因：** 数据库字符集不是 utf8mb4

**解决：** 
```sql
ALTER DATABASE seoapi CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

## 重要提醒

1. **备份数据：** 在执行任何数据库操作前，务必备份现有数据
2. **停止服务：** 初始化期间停止应用服务，避免数据冲突
3. **验证结果：** 初始化后运行诊断脚本验证
4. **修改密码：** 首次登录后立即修改默认管理员密码
