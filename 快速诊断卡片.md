# 快速诊断卡片 - 小旋风配置

## 🚨 问题：请求API失败，返回空内容

---

## ⚡ 30秒快速检查

### 1️⃣ 服务是否运行？

```bash
curl http://localhost:8000/health
```

✅ 返回 `{"status":"ok"}` → 服务正常  
❌ 无响应 → 启动服务：`cd seoapi-nodejs && node server.js`

---

### 2️⃣ 小旋风配置是否正确？

**复制这个配置（替换 API Key）：**

```
API名称: Claude AI 伪原创
API地址: http://localhost:8000/api/rewrite
post格式: txt={word}&mode=seo_fast&sim=1
Authorization: sk_f7747efeefece9fbeb79bfecd825e01ff427c2f343acd398f6f3713469906d00
成功标志: errcode=0
返回错误的字段: errmsg
伪原创后的内容字段: data
```

**⚠️ 注意事项：**
- ✅ 使用 `{word}` 不是 `[content]`
- ✅ API 地址只有一个斜杠 `/api/rewrite`
- ✅ Authorization 字段名正确
- ✅ 成功标志是 `errcode=0` 不带引号

---

### 3️⃣ API Key 是否正确？

**查看你的 API Key：**

打开文件：`seoapi-nodejs/data/users.json`

```json
{
  "admin": {
    "apiKey": "sk_xxxxx..."  ← 复制这个值到小旋风的 Authorization
  }
}
```

---

### 4️⃣ Claude API Key 是否配置？

**检查配置：**

```json
{
  "admin": {
    "claudeApiKey": "sk-xxxxx...",  ← 必须配置
    "claudeBaseURL": "https://api123.icu"  ← 必须正确
  }
}
```

❌ 如果 `claudeApiKey` 为空 → 在 Web 界面配置（http://localhost:8000）

## 🔧 常见错误速查

| 小旋风提示 | 原因 | 解决方案 |
|-----------|------|---------|
| 返回空内容 | 服务未启动 | `node server.js` |
| 返回空内容 | API 地址错误 | 检查是否有双斜杠 `//` |
| 无效的 apikey | API Key 错误 | 检查 users.json |
| 用户未配置 Claude API Key | 未配置 Claude | Web 界面配置 |
| 504 超时 | 文本太长或模型太慢 | 使用 Haiku 或 Sonnet |

## 🧪 快速测试命令

**PowerShell 测试（替换 API Key）：**

```powershell
$apiKey = "sk_f7747efeefece9fbeb79bfecd825e01ff427c2f343acd398f6f3713469906d00"
$headers = @{"Authorization"=$apiKey; "Content-Type"="application/x-www-form-urlencoded"}
$body = "txt=人工智能技术正在快速发展&mode=seo_fast&sim=1"
Invoke-WebRequest -Uri "http://localhost:8000/api/rewrite" -Method POST -Headers $headers -Body $body -UseBasicParsing
```

**预期结果：**
```json
{
  "errcode": "0",
  "data": "改写后的文本..."
}
```
