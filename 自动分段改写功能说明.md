# 自动分段改写功能说明

## 功能概述

系统现在支持**自动分段改写**超长文本，无需手动分割！

当文本超过 2000 字符时，系统会自动：
1. 智能分段（按段落和句子分割）
2. 分别改写每一段
3. 自动合并结果

## 工作原理

### 触发条件

```
文本长度 > 2000 字符 → 自动启用分段模式
文本长度 ≤ 2000 字符 → 标准处理模式
```

### 分段策略

**智能分段算法：**
1. 优先按段落分割（双换行 `\n\n`）
2. 如果段落太长，按句子分割（句号、问号、感叹号）
3. 每段目标大小：1800 字符
4. 避免截断句子，保持语义完整

**示例：**
```
原文 5000 字符
    ↓ 智能分段
段落 1: 1750 字符
段落 2: 1800 字符
段落 3: 1450 字符
    ↓ 分别改写
改写 1: 1900 字符
改写 2: 2000 字符
改写 3: 1600 字符
    ↓ 合并
最终结果: 5500 字符
```

### 处理流程

```
1. 检测文本长度
   ↓
2. 如果 > 2000 字符
   ↓
3. 智能分段（1800 字符/段）
   ↓
4. 循环处理每一段：
   - 调用 AI 改写
   - 记录耗时和 Token
   - 保存结果
   ↓
5. 合并所有段落（用 \n\n 连接）
   ↓
6. 返回完整结果
```

## 配置参数


## 注意事项

### 1. 上下文连贯性

**影响：** 分段处理可能影响段落间的连贯性

**原因：** AI 无法看到完整的上下文

**建议：**
- 短文本（< 2000 字符）：使用标准模式
- 长文本（> 2000 字符）：接受轻微的连贯性损失

### 2. 处理时间

**总时间 = 段数 × 单段时间**

**示例：**
- 3 段 × 25 秒 = 75 秒
- 5 段 × 25 秒 = 125 秒

**建议：** 超过 10000 字符的文本建议分批处理

### 3. 成本

**Token 使用量与标准模式基本相同**

但会有轻微增加（约 5-10%），因为：
- 每段都需要提示词
- 段落间可能有重复内容

### 4. 小旋风兼容性

**分段模式不适合小旋风：**
- 响应体可能超过 1500 字符
- 建议小旋风仍然限制采集长度

## 常见问题

### Q1: 分段会影响改写质量吗？

**A:** 会有轻微影响，主要是段落间的连贯性。但对于超长文本，这是唯一的解决方案。

### Q2: 可以手动控制是否分段吗？

**A:** 目前是自动的（> 2000 字符自动分段）。如果需要手动控制，可以添加参数。

### Q3: 分段大小可以调整吗？

**A:** 可以，修改 `CHUNK_SIZE` 常量（推荐 1800 字符）。

### Q4: 为什么不是 2000 字符一段？

**A:** 留 200 字符的安全余量，避免接近超时边界。

### Q5: 分段模式会增加成本吗？

**A:** Token 使用量基本相同，但调用次数增加。总成本增加约 5-10%。

## 技术细节

### 智能分段算法

```javascript
function smartSplitText(text, chunkSize) {
    // 1. 按段落分割（双换行）
    const paragraphs = text.split(/\n\n+/);
    
    // 2. 累积段落直到接近 chunkSize
    let currentChunk = '';
    const chunks = [];
    
    for (const paragraph of paragraphs) {
        // 3. 如果单个段落太长，按句子分割
        if (paragraph.length > chunkSize) {
            // 按句子分割
            const sentences = paragraph.split(/([。！？.!?]+)/);
            // 累积句子...
        } else {
            // 4. 累积段落
            if (currentChunk.length + paragraph.length > chunkSize) {
                chunks.push(currentChunk);
                currentChunk = paragraph;
            } else {
                currentChunk += '\n\n' + paragraph;
            }
        }
    }
    
    return chunks;
}
```
