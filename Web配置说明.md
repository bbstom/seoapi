# Web 界面配置改写模式 - 零代码教程

## 🎉 好消息！

现在你可以直接在浏览器里管理改写模式，完全不需要写代码！

## 🚀 快速开始

### 步骤 1: 启动服务

```bash
# Windows
start.bat

# Linux
./start.sh
```

### 步骤 2: 打开浏览器

访问：http://localhost:8000

### 步骤 3: 点击"模式管理"标签

你会看到所有现有的改写模式

## ➕ 添加新模式

### 图文教程

1. **点击"+ 添加新模式"按钮**
   
2. **填写表单：**

   - **模式 ID**（必填）
     ```
     例如：xiaohongshu
     只能用英文小写字母和下划线
     这个 ID 用于 API 调用
     ```
   
   - **模式名称**（必填）
     ```
     例如：小红书风格
     这是显示在界面上的名字
     ```
   
   - **简短描述**（必填）
     ```
     例如：适合小红书平台发布
     一句话说明这个模式的用途
     ```
   
   - **反 AI 检测模式**（可选）
     ```
     如果这个模式可以降低 AI 检测风险，勾选这个
     ```
   
   - **改写指令**（必填，最重要！）
     ```
     告诉 AI 如何改写文本
     
     例如：
     请将以下文本改写成小红书风格：
     1. 使用emoji表情 ✨💕
     2. 分段清晰，每段不超过3行
     3. 添加个人体验和感受
     4. 使用"姐妹们"、"真的"等口语
     5. 结尾加互动问题
     ```

3. **点击"保存模式"**

4. **完成！** 新模式会立即出现在列表中

## ✏️ 编辑现有模式

1. 找到要编辑的模式
2. 点击"编辑"按钮
3. 修改内容
4. 点击"保存模式"

**注意：** 内置模式（带"内置"标签的）可以编辑，但建议保留原样

## 🗑️ 删除模式

1. 找到要删除的模式
2. 点击"删除"按钮
3. 确认删除

**注意：** 内置模式不能删除，只能编辑

## 🧪 测试模式

1. 找到要测试的模式
2. 点击"测试"按钮
3. 自动跳转到"文本改写"页面
4. 已经选好了这个模式
5. 输入文本，点击"开始改写"

## 💡 编写改写指令的技巧

### 技巧 1: 使用编号列表

✅ 好的例子：
```
请改写，要求：
1. 第一点要求
2. 第二点要求
3. 第三点要求
```

❌ 不好的例子：
```
请改写这段文字
```

### 技巧 2: 给出具体例子

✅ 好的例子：
```
请改写成口语化风格：
1. 使用"其实"、"不过"、"比如"等词
2. 像这样："其实这个方法挺简单的"
3. 而不是："该方法较为简单"
```

### 技巧 3: 说明不要做什么

```
请改写，要求：
1. 不要使用过于正式的词汇
2. 不要使用专业术语
3. 避免长句子（超过30字）
```

### 技巧 4: 指定目标受众

```
请改写，目标读者是：
- 年龄：25-35岁
- 职业：互联网从业者
- 需求：快速获取信息
要求：专业但不枯燥，有趣但不低俗
```

## 🔄 配置文件位置

虽然你不需要手动编辑，但配置文件保存在：

```
seoapi-nodejs/config/rewrite-modes.js
```

Web 界面的所有修改都会自动保存到这个文件。

## ❓ 常见问题

### Q1: 保存后需要重启服务吗？

**A:** 不需要！保存后立即生效。

### Q2: 可以修改内置模式吗？

**A:** 可以编辑，但不能删除。建议保留内置模式，添加新的自定义模式。

### Q3: 模式 ID 有什么要求？

**A:** 
- 只能使用小写英文字母和下划线
- 不能有空格和特殊字符
- 例如：`my_mode`、`xiaohongshu`、`tech_blog`

### Q4: 改写指令写多长合适？

**A:** 
- 建议 3-10 条要求
- 每条要求简洁明确
- 总长度不超过 500 字

### Q5: 如何知道模式效果好不好？

**A:** 
1. 点击"测试"按钮
2. 输入示例文本
3. 查看改写结果
4. 如果不满意，回去编辑指令

### Q6: 可以复制别人的模式吗？

**A:** 
可以！步骤：
1. 查看别人的模式配置
2. 点击"添加新模式"
3. 复制粘贴改写指令
4. 修改模式 ID 和名称
5. 保存

### Q7: 删除模式会影响已经使用的吗？

**A:** 
- 如果小旋风正在使用这个模式，删除后会报错
- 建议先停止使用，再删除
- 或者编辑模式而不是删除

## 🎓 进阶技巧

### 技巧 1: 创建模式组

为不同场景创建一组模式：

```
SEO 组：
- seo_standard (标准 SEO)
- seo_long (长文 SEO)
- seo_short (短文 SEO)

自媒体组：
- xiaohongshu (小红书)
- douyin (抖音)
- weibo (微博)
```

### 技巧 2: 版本管理

为同一场景创建不同版本：

```
- xiaohongshu_v1 (基础版)
- xiaohongshu_v2 (进阶版)
- xiaohongshu_v3 (专业版)
```

### 技巧 3: A/B 测试

创建两个相似的模式，测试哪个效果更好：

```
- humanlike_a (方案 A)
- humanlike_b (方案 B)
```
