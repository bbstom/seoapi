# 上下文重叠分段策略说明

## 问题：简单分段的语义问题

### 原始方案的问题

**简单分段：**
```
段落 1: "...人工智能的发展历程。"
段落 2: "深度学习技术的突破..."
```

**问题：**
- ❌ AI 看不到前文，不知道"深度学习"是在讨论什么
- ❌ 段落间缺乏连贯性
- ❌ 改写风格可能不一致
- ❌ 可能重复或遗漏信息

---

## 解决方案：上下文重叠

### 改进后的策略

**带上下文的分段：**
```
段落 1:
  文本: "...人工智能的发展历程。"
  后文参考: "深度学习技术的突破..."

段落 2:
  前文参考: "...人工智能的发展历程。"
  文本: "深度学习技术的突破..."
  后文参考: "在计算机视觉领域..."

段落 3:
  前文参考: "...深度学习技术的突破..."
  文本: "在计算机视觉领域..."
```

**优势：**
- ✅ AI 能看到前后文
- ✅ 保持段落间的连贯性
- ✅ 改写风格更一致
- ✅ 避免重复或遗漏

---

## 技术实现

### 1. 上下文重叠大小

```javascript
const OVERLAP_SIZE = 200; // 200 字符的上下文
```

**说明：**
- 前文参考：取上一段的最后 200 字符
- 后文参考：取下一段的前 200 字符
- 200 字符约等于 100 个中文字或 1-2 个段落

### 2. 提示词构建

**标准提示词：**
```
请改写以下文本...

原文：
[文本内容]

改写后的文本：
```

**带上下文的提示词：**
```
请改写以下文本...

【重要提示】这是一篇长文章的一部分，请注意保持与前后文的连贯性：

前文参考（仅供理解上下文，不要改写）：
"...人工智能的发展历程。"

后文参考（仅供理解上下文，不要改写）：
"在计算机视觉领域..."

原文：
深度学习技术的突破使得人工智能在多个领域取得了显著进展。

改写后的文本：
```

### 3. 分段对象结构

```javascript
{
    text: "要改写的文本",
    previousContext: "前文参考（200字符）",
    nextContext: "后文参考（200字符）"
}
```

## 效果对比

### 示例文本

**原文：**
```
人工智能技术正在改变世界。深度学习是其中的关键技术。
它在图像识别领域取得了突破。这项技术已经广泛应用。
```

### 不带上下文的改写

**段落 1：**
```
输入: "人工智能技术正在改变世界。深度学习是其中的关键技术。"
输出: "AI 科技正在重塑全球格局。深度学习扮演着核心角色。"
```

**段落 2：**
```
输入: "它在图像识别领域取得了突破。这项技术已经广泛应用。"
输出: "该方法在视觉识别方面实现了飞跃。相关应用遍地开花。"
         ↑ 不知道"该方法"指什么
```

**问题：** 段落 2 不知道"它"和"这项技术"指的是什么。

### 带上下文的改写

**段落 1：**
```
输入: "人工智能技术正在改变世界。深度学习是其中的关键技术。"
后文: "它在图像识别领域取得了突破..."
输出: "AI 科技正在重塑全球格局。深度学习扮演着核心角色。"
```

**段落 2：**
```
前文: "...深度学习是其中的关键技术。"
输入: "它在图像识别领域取得了突破。这项技术已经广泛应用。"
输出: "深度学习在视觉识别方面实现了飞跃。这一技术已经遍地开花。"
         ↑ 明确指出是"深度学习"
```

**优势：** 段落 2 知道"它"指的是"深度学习"，改写更准确。

## 参数调整

### 上下文大小

```javascript
const OVERLAP_SIZE = 200; // 可调整
```

**推荐值：**
- 最小：100 字符（约 50 个中文字）
- 推荐：200 字符（约 100 个中文字）
- 最大：300 字符（约 150 个中文字）

**权衡：**
- 太小：上下文不足，连贯性差
- 太大：增加 Token 消耗，处理时间长
- 推荐：200 字符是平衡点

### 分段大小

```javascript
const CHUNK_SIZE = 1800; // 可调整
```

**考虑因素：**
- 上下文开销：200 × 2 = 400 字符
- 实际文本：1800 字符
- 总输入：1800 + 400 = 2200 字符
- 处理时间：约 25-30 秒 ✅

## 使用建议

### 何时使用上下文重叠？

**推荐使用：**
- ✅ 技术文章（需要保持术语一致性）
- ✅ 故事叙述（需要保持人物和情节连贯）
- ✅ 学术论文（需要保持逻辑连贯）
- ✅ 长篇内容（> 3000 字符）

**可以不用：**
- ⚪ 新闻报道（段落相对独立）
- ⚪ 列表内容（项目间独立）
- ⚪ 短文本（< 2000 字符，不需要分段）

### 配置建议

**最佳配置：**
```javascript
const CHUNK_SIZE = 1800;      // 每段大小
const OVERLAP_SIZE = 200;     // 上下文重叠
const TRIGGER_SIZE = 2000;    // 触发分段的阈值
```

**保守配置（更安全）：**
```javascript
const CHUNK_SIZE = 1500;
const OVERLAP_SIZE = 250;
const TRIGGER_SIZE = 1800;
```

**激进配置（更快）：**
```javascript
const CHUNK_SIZE = 2000;
const OVERLAP_SIZE = 150;
const TRIGGER_SIZE = 2500;
```

## 总结

### 改进点

1. ✅ **添加上下文重叠** - 每段都能看到前后文
2. ✅ **智能提示词** - 明确告诉 AI 这是长文章的一部分
3. ✅ **保持连贯性** - 段落间的过渡更自然
4. ✅ **避免语义错误** - AI 知道代词和术语的指代

### 成本

- Token 使用增加约 15%
- 处理时间增加约 10%（因为输入更长）
- 但改写质量显著提升 ✅

### 建议

**对于超长文本（> 2000 字符）：**
- 强烈建议使用上下文重叠
- 200 字符的重叠是最佳平衡点
- 成本增加是值得的

**对于短文本（< 2000 字符）：**
- 不需要分段
- 使用标准处理模式即可
