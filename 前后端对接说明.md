# 前后端对接说明

## 📋 架构概述

本项目采用**前后端分离**架构：
- **后端**: Node.js + Express（端口 8000）
- **前端**: React + Vite（开发端口 3000，生产环境构建到 public 目录）

---

## 🔄 对接方式

### 开发环境

#### 1. 前端开发服务器（Vite）
- **端口**: 3000
- **启动命令**: `npm run dev`（在 frontend 目录）
- **配置文件**: `frontend/vite.config.js`

```javascript
server: {
  port: 3000,
  proxy: {
    '/api': {
      target: 'http://localhost:8000',  // 代理到后端
      changeOrigin: true
    }
  }
}
```

#### 2. 后端服务器（Express）
- **端口**: 8000
- **启动命令**: `node server.js` 或 `start.bat`
- **静态文件**: 服务 `public` 目录

```javascript
// server.js
app.use(express.static('public'));
```

#### 3. 开发流程

```
前端请求 (localhost:3000)
    ↓
Vite 代理 (/api/*)
    ↓
后端 API (localhost:8000/api/*)
    ↓
返回 JSON 数据
```

**示例：**
```javascript
// 前端代码
axios.get('/api/user/profile')
// 实际请求: http://localhost:3000/api/user/profile
// Vite 代理到: http://localhost:8000/api/user/profile
```

#### 1. 前端构建
```bash
cd frontend

```

**构建配置**（`frontend/vite.config.js`）:
```javascript
build: {
  outDir: '../public',      // 构建到父目录的 public 文件夹
  emptyOutDir: true,        // 清空目标目录
  rollupOptions: {
    output: {
      manualChunks: {
        vendor: ['react', 'react-dom'],
        icons: ['lucide-react']
      }
    }
  }
}
```

**构建结果**:
```

├── index.html           # 前端入口
├── assets/
│   ├── index-xxx.js    # 打包后的 JS
│   └── index-xxx.css   # 打包后的 CSS
└── ...
```

#### 2. 后端服务
```javascript
// server.js
app.use(express.static('public'));  // 服务前端静态文件

// API 路由
app.post('/api/rewrite', ...);
app.get('/api/user/profile', ...);
// ... 其他 API
```

#### 3. 生产流程

```
用户访问 (http://your-domain.com)
    ↓
Express 静态文件服务
    ↓
返回 public/index.html
    ↓
浏览器加载 React 应用
    ↓
React 发起 API 请求 (/api/*)
    ↓
Express API 路由处理
    ↓
返回 JSON 数据
```

## 🔌 API 调用方式

### 前端配置

#### 1. API 基础路径
```javascript
// frontend/src/App.jsx
const API_BASE = '/api';
```

#### 2. Axios 拦截器（自动添加认证）
```javascript
axios.interceptors.request.use(
  (config) => {
    const token = localStorage.getItem('sessionToken');
    if (token) {
      config.headers['x-session-token'] = token;
    }
    return config;
  },
  (error) => {
    return Promise.reject(error);
  }
);
```

#### 3. API 调用示例
```javascript
// 登录
const response = await axios.post('/api/login', {
  username: 'admin',
  password: 'admin123'
});

// 获取用户信息
const response = await axios.get('/api/user/profile');

// 文本改写
const response = await axios.post('/api/rewrite', {
  txt: '要改写的文本',
  mode: 'humanlike',
  model: 'claude-sonnet-4-5-20250929'
});
```

## 🔒 认证机制

### 会话认证流程

```
1. 用户登录
   POST /api/login
   { username, password }
   ↓
2. 后端验证
   - 检查用户名密码
   - 生成 session token
   - 存储到 sessions 表
   ↓
3. 返回 token
   { success: true, token: 'xxx' }
   ↓
4. 前端存储
   localStorage.setItem('sessionToken', token)
   ↓
5. 后续请求
   axios 拦截器自动添加
   headers: { 'x-session-token': token }
   ↓
6. 后端验证
   - 检查 token 是否有效
   - 检查是否过期
   - 返回用户信息
```

### 认证中间件

```javascript
// 后端 server.js
async function requireAuth(req, res, next) {
  const token = req.headers['x-session-token'];
  
  if (!token) {
    return res.status(401).json({ 
      success: false, 
      error: '未登录' 
    });
  }
  
  // 验证 token
  const session = await sessionManager.getSession(token);
  if (!session) {
    return res.status(401).json({ 
      success: false, 
      error: '会话已过期' 
    });
  }
  
  req.user = session.user;
  next();
}

// 使用中间件
app.get('/api/user/profile', requireAuth, async (req, res) => {
  // req.user 已包含用户信息
  res.json({ success: true, user: req.user });
});
```

## 🌐 跨域处理

### 开发环境
- **前端**: Vite 代理（`vite.config.js`）
- **后端**: CORS 中间件

```javascript
// server.js
const cors = require('cors');
app.use(cors());  // 允许所有跨域请求
```

### 生产环境
- **无需跨域**: 前后端同域（都在 8000 端口）
- **如使用 Nginx**: 配置反向代理

```nginx
server {
    listen 80;
    server_name your-domain.com;
    
    location / {
        proxy_pass http://localhost:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

## 🔧 常见问题

### Q1: 开发时前端无法访问后端 API？

**A**: 检查以下几点：
1. 后端是否启动（端口 8000）
2. Vite 代理配置是否正确
3. API 路径是否以 `/api` 开头

### Q2: 生产环境前端页面空白？

**A**: 检查以下几点：
1. 前端是否已构建（`npm run build`）
2. `public` 目录是否有 `index.html`
3. 后端是否正确服务静态文件

### Q3: API 请求返回 401 未授权？

**A**: 检查以下几点：
1. 是否已登录
2. `localStorage` 中是否有 `sessionToken`
3. token 是否过期（24小时）

### Q4: 如何查看 API 请求？

**A**: 
- **浏览器**: F12 → Network 标签
- **后端日志**: 查看控制台输出
- **数据库日志**: 查看 `api_logs` 表

- 前端: http://localhost:3000（Vite 开发服务器）
- 后端: http://localhost:8000（Express 服务器）

- `server.js` - 后端主文件
