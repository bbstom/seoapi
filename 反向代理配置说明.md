# 反向代理配置说明

## 🔍 问题说明

### 错误信息

```
ValidationError: The 'X-Forwarded-For' header is set but the Express 'trust proxy' setting is false (default).
```

### 问题原因

当你的 Node.js 应用部署在反向代理（如 Nginx、Apache、Caddy）后面时：

1. **客户端** → **反向代理（Nginx）** → **Node.js 应用**
2. Nginx 会添加 `X-Forwarded-For` 头来传递真实客户端 IP
3. 但 Express 默认不信任这个头（`trust proxy = false`）
4. `express-rate-limit` 需要正确的客户端 IP 来限流
5. 导致冲突和错误

### 影响

- ❌ 速率限制无法正确识别用户
- ❌ 所有请求可能被识别为同一个 IP（代理服务器 IP）
- ❌ 日志记录的 IP 地址不准确
- ❌ 无法正确实施访问控制

## ✅ 解决方案

### 方案 1：根据环境自动配置（推荐，已实现）

已在 `server.js` 中添加：

```javascript
// 配置 trust proxy（根据环境自动调整）
if (process.env.NODE_ENV === 'production') {
    // 生产环境：只信任 1 层代理（更安全）
    app.set('trust proxy', 1);
} else {
    // 开发环境：信任本地代理
    app.set('trust proxy', 'loopback');
}
```

**配置环境变量：**

在 `.env` 文件中设置：

```env
# 开发环境（本地测试）
NODE_ENV=development

# 生产环境（部署到服务器）
NODE_ENV=production
```

**适用场景：**
- ✅ 自动适配开发和生产环境
- ✅ 开发环境不会触发安全警告
- ✅ 生产环境更安全（只信任 1 层代理）
- ✅ 推荐使用

### 方案 2：指定信任的代理数量

如果你知道代理层级数量：

```javascript
// 信任 1 层代理（最常见）
app.set('trust proxy', 1);

// 信任 2 层代理（例如：CDN + Nginx）
app.set('trust proxy', 2);
```

### 方案 3：指定信任的代理 IP

如果你知道代理服务器的 IP：

```javascript
// 信任特定 IP
app.set('trust proxy', '127.0.0.1');

// 信任多个 IP
app.set('trust proxy', ['127.0.0.1', '192.168.1.1']);

// 信任 IP 段
app.set('trust proxy', 'loopback, 192.168.1.0/24');
```

## 🔧 Nginx 配置

### 标准配置

```nginx
server {
    listen 80;
    server_name api.vpno.eu.org;

    location / {
        proxy_pass http://localhost:8000;
        
        # 传递真实客户端 IP
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        
        # 传递主机信息
        proxy_set_header Host $host;
        
        # WebSocket 支持（如果需要）
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_cache_bypass $http_upgrade;
        
        # 超时设置
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;
    }
}
```

### HTTPS 配置

```nginx
server {
    listen 443 ssl http2;
    server_name api.vpno.eu.org;

    ssl_certificate /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;

    location / {
        proxy_pass http://localhost:8000;
        
        # 传递真实客户端 IP
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto https;
        
        # 传递主机信息
        proxy_set_header Host $host;
        
        # 超时设置（针对 Claude API 可能的长响应时间）
        proxy_connect_timeout 90s;
        proxy_send_timeout 90s;
        proxy_read_timeout 90s;
    }
}

# HTTP 重定向到 HTTPS
server {
    listen 80;
    server_name api.vpno.eu.org;
    return 301 https://$server_name$request_uri;
}
```

## 📊 Trust Proxy 选项对比

| 选项 | 说明 | 适用场景 | 安全性 |
|------|------|---------|--------|
| `false` | 不信任代理（默认） | 直接暴露服务 | ⭐⭐⭐⭐⭐ |
| `true` | 信任所有代理 | 简单部署 | ⭐⭐⭐ |
| `1` | 信任 1 层代理 | Nginx/Apache | ⭐⭐⭐⭐ |
| `2` | 信任 2 层代理 | CDN + Nginx | ⭐⭐⭐⭐ |
| `'127.0.0.1'` | 信任特定 IP | 本地代理 | ⭐⭐⭐⭐⭐ |
| `['127.0.0.1', '192.168.1.1']` | 信任多个 IP | 多个代理 | ⭐⭐⭐⭐⭐ |

## ⚠️ 安全注意事项

### 1. 不要盲目信任所有代理

❌ **不安全：**
```javascript
app.set('trust proxy', true);  // 信任所有代理
```

如果攻击者可以伪造 `X-Forwarded-For` 头，可能绕过速率限制。

✅ **更安全：**
```javascript
app.set('trust proxy', 1);  // 只信任第一层代理
```

### 2. 使用白名单

✅ **最安全：**
```javascript
// 只信任已知的代理 IP
app.set('trust proxy', ['127.0.0.1', '::1']);
```

### 3. 验证代理配置

确保 Nginx 正确设置了 `X-Forwarded-For`：

```nginx
# 正确：追加客户端 IP
proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;

# 错误：直接覆盖（可能被伪造）
proxy_set_header X-Forwarded-For $remote_addr;
```

## 🔍 常见问题

### Q1: 为什么需要 trust proxy？

**A:** 当使用反向代理时：
- 客户端 → Nginx (IP: 1.2.3.4) → Node.js
- Node.js 看到的 IP 是 127.0.0.1（Nginx 的 IP）
- 真实客户端 IP 在 `X-Forwarded-For` 头中
- 需要告诉 Express 信任这个头

### Q2: trust proxy = true 安全吗？

**A:** 取决于部署环境：
- ✅ 安全：如果只有你控制的 Nginx 可以访问 Node.js
- ❌ 不安全：如果 Node.js 直接暴露在公网

**推荐做法：**
```javascript
// 生产环境：只信任 1 层代理
app.set('trust proxy', 1);

// 或指定代理 IP
app.set('trust proxy', '127.0.0.1');
```

### Q3: 如何验证配置是否正确？

**A:** 检查日志中的 IP 地址：

```javascript
app.use((req, res, next) => {
    console.log('req.ip:', req.ip);  // 应该是真实客户端 IP
    next();
});
```

### Q4: 使用 CDN 需要特殊配置吗？

**A:** 是的，CDN 会增加一层代理：

```javascript
// Cloudflare + Nginx = 2 层代理
app.set('trust proxy', 2);

// 或者信任 Cloudflare IP 段
app.set('trust proxy', [
    '127.0.0.1',
    '173.245.48.0/20',
    '103.21.244.0/22',
    // ... 其他 Cloudflare IP
]);
```

### Q5: 宝塔面板需要特殊配置吗？

**A:** 宝塔面板使用 Nginx，配置方法相同：

1. 在 Node.js 应用中设置 `trust proxy`
2. 在宝塔的反向代理配置中确保传递了正确的头
3. 重启 Node.js 应用和 Nginx
