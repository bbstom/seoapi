Trust Proxy 警告快速修复
========================================

错误信息：
ValidationError: The Express 'trust proxy' setting is true,
which allows anyone to trivially bypass IP-based rate limiting.

========================================
✅ 已修复（设置环境变量即可）
========================================

修复方案：根据环境自动配置
- 开发环境：trust proxy = loopback
- 生产环境：trust proxy = 1

========================================
🔧 本地开发（Windows）
========================================

方法 1：创建 .env 文件（推荐）
echo NODE_ENV=development > .env
node server.js

方法 2：PowerShell 设置
$env:NODE_ENV="development"
node server.js

方法 3：CMD 设置
set NODE_ENV=development
node server.js

========================================
🚀 服务器部署（Linux）
========================================

方法 1：创建 .env 文件（推荐）
echo "NODE_ENV=production" > .env
pm2 restart seoapi

方法 2：PM2 启动时设置
NODE_ENV=production pm2 start server.js --name seoapi

方法 3：PM2 配置文件
# 创建 ecosystem.config.js
pm2 start ecosystem.config.js --env production

========================================
🔍 环境差异
========================================

开发环境 (NODE_ENV=development)：
- Trust Proxy: loopback
- 只信任本地回环地址
- 适合本地测试
- 不会触发警告

生产环境 (NODE_ENV=production)：
- Trust Proxy: 1
- 只信任 1 层代理（Nginx）
- 适合服务器部署
- 更安全可靠
