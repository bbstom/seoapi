# 环境配置说明

## 📋 环境变量配置

### NODE_ENV 环境变量

`NODE_ENV` 用于区分开发环境和生产环境，影响以下行为：

| 环境 | NODE_ENV 值 | Trust Proxy | 调试信息 | 适用场景 |
|------|------------|-------------|---------|---------|
| 开发环境 | `development` | loopback | 显示 | 本地测试 |
| 生产环境 | `production` | 1 | 隐藏 | 服务器部署 |

## 🔧 配置方法

### 方法 1：使用 .env 文件（推荐）

创建或编辑 `.env` 文件：

**开发环境（本地测试）：**
```env
# Claude API 配置
CLAUDE_API_KEY=your_api_key_here
CLAUDE_BASE_URL=https://api.api123.icu

# 服务配置
PORT=8000

# 环境配置
NODE_ENV=development
```

**生产环境（服务器部署）：**
```env
# Claude API 配置
CLAUDE_API_KEY=your_api_key_here
CLAUDE_BASE_URL=https://api.api123.icu

# 服务配置
PORT=8000

# 环境配置
NODE_ENV=production
```

### 方法 2：命令行设置

**Windows (PowerShell)：**
```powershell
# 开发环境
$env:NODE_ENV="development"
node server.js

# 生产环境
$env:NODE_ENV="production"
node server.js
```

**Windows (CMD)：**
```cmd
# 开发环境
set NODE_ENV=development
node server.js

# 生产环境
set NODE_ENV=production
node server.js
```

**Linux/Mac：**
```bash
# 开发环境
NODE_ENV=development node server.js

# 生产环境
NODE_ENV=production node server.js
```

### 方法 3：PM2 配置

创建 `ecosystem.config.js`：

```javascript
module.exports = {
  apps: [{
    name: 'seoapi',
    script: './server.js',
    env: {
      NODE_ENV: 'development',
      PORT: 8000
    },
    env_production: {
      NODE_ENV: 'production',
      PORT: 8000
    }
  }]
};
```

**启动命令：**
```bash
# 开发环境
pm2 start ecosystem.config.js

# 生产环境
pm2 start ecosystem.config.js --env production
```

## 🎯 环境差异

### 开发环境 (NODE_ENV=development)

**Trust Proxy 配置：**
```javascript
app.set('trust proxy', 'loopback');
// 只信任本地回环地址（127.0.0.1, ::1）
```

**特点：**
- ✅ 适合本地测试
- ✅ 不会触发安全警告
- ✅ 显示详细的调试信息
- ✅ 返回 `_meta` 元数据

**适用场景：**
- 本地开发和测试
- 直接运行 `node server.js`
- 不使用反向代理

### 生产环境 (NODE_ENV=production)

**Trust Proxy 配置：**
```javascript
app.set('trust proxy', 1);
// 只信任 1 层代理（Nginx/Apache）
```

**特点：**
- ✅ 更安全（只信任 1 层代理）
- ✅ 正确识别客户端 IP
- ✅ 速率限制正常工作
- ✅ 隐藏敏感调试信息
- ✅ 不返回 `_meta` 元数据（除非 debug=1）

**适用场景：**
- 服务器部署
- 使用 Nginx/Apache 反向代理
- 使用 PM2 管理进程
- 对外提供服务

## ⚠️ 常见问题

### Q1: 本地测试时出现 trust proxy 警告？

**A:** 设置 `NODE_ENV=development`

```env
NODE_ENV=development
```

或者在启动时设置：
```bash
NODE_ENV=development node server.js
```

### Q2: 生产环境速率限制不工作？

**A:** 确保设置了 `NODE_ENV=production`

```env
NODE_ENV=production
```

### Q3: 如何查看当前环境？

**A:** 查看启动日志：

```bash
# 应该显示
Trust proxy: 1 (生产环境)
# 或
Trust proxy: loopback (开发环境)
```

### Q4: 可以不设置 NODE_ENV 吗？

**A:** 可以，默认为开发环境

如果不设置 `NODE_ENV`，系统会使用开发环境配置（`trust proxy: loopback`）

### Q5: 使用宝塔面板如何设置？

**A:** 在 Node 项目设置中添加环境变量

1. 进入宝塔面板
2. 找到 Node 项目管理
3. 点击项目设置
4. 添加环境变量：`NODE_ENV=production`
5. 重启项目

## 🚀 部署建议

```bash

# 2. 启动服务
npm start
```

### 服务器部署

**方法 1：使用 .env 文件**
```bash
# 1. 创建 .env 文件
echo "NODE_ENV=production" > .env
echo "PORT=8000" >> .env

# 2. 使用 PM2 启动
pm2 start server.js --name seoapi
```

**方法 2：使用 PM2 配置文件**
```bash
# 1. 创建 ecosystem.config.js
# 2. 启动

```

**方法 3：直接设置环境变量**
```bash
NODE_ENV=production pm2 start server.js --name seoapi
```

## 🔐 安全最佳实践

### 1. 生产环境必须设置 NODE_ENV=production

```env
NODE_ENV=production
```

**原因：**
- 更安全的 trust proxy 配置
- 隐藏敏感调试信息
- 优化性能

### 2. 不要在生产环境使用 trust proxy = true

❌ **不安全：**
```javascript
app.set('trust proxy', true);  // 任何人都可以伪造 IP
```

✅ **安全：**
```javascript
app.set('trust proxy', 1);  // 只信任 1 层代理
```

### 3. 使用防火墙限制访问

确保只有 Nginx 可以访问 Node.js 端口：

```bash
# 只允许本地访问 8000 端口
iptables -A INPUT -p tcp --dport 8000 -s 127.0.0.1 -j ACCEPT
iptables -A INPUT -p tcp --dport 8000 -j DROP
```

### 4. 定期更新依赖

```bash
npm update
npm audit fix
```
