在生产服务器上执行以下命令：

```bash

pm2 stop seoapi

mysql -h 172.16.254.100 -u root -pUbuntu123! -e "DROP DATABASE IF EXISTS seoapi; CREATE DATABASE seoapi CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"

node scripts/init-database-from-sql.js

# 5. 验证结果
node scripts/diagnose-database.js

# 6. 启动服务
pm2 start seoapi

# 7. 查看日志确认无错误
pm2 logs seoapi --lines 50
```

## 常见问题

### Q: 会丢失现有数据吗？

A: 是的，这个方案会重建数据库。如果有重要数据，请先备份：

```bash
mysqldump -h 172.16.254.100 -u root -pUbuntu123! seoapi > backup_$(date +%Y%m%d_%H%M%S).sql
```

### Q: 可以保留数据吗？

A: 如果要保留数据，可以直接导入 seoapi2.sql：

```bash
mysql -h 172.16.254.100 -u root -pUbuntu123! seoapi < seoapi2.sql
```

但这会覆盖所有现有数据。

### Q: 初始化后还是报错怎么办？

A: 
1. 确认使用的是 `init-database-from-sql.js` 而不是旧脚本
2. 运行 `node scripts/diagnose-database.js` 查看详细信息
3. 检查错误日志：`tail -f /www/wwwlogs/pm2/seoapi-error.log`

### Q: 默认管理员密码是什么？

A: 
- 用户名：`admin`
