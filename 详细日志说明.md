## 🔍 常见错误日志分析

### 错误 1：文本为空

```
[请求 xxx] ❌ 错误: 文本为空
```

**原因：** 小旋风没有正确传递文本参数

**排查：**
1. 检查小旋风配置中的 `post格式`
2. 确认使用 `txt={word}` 而不是 `text={word}`
3. 查看 `请求体参数` 中的 `text_length` 是否为 0

**解决：**
```
post格式: txt={word}&mode=seo_fast&sim=1
```

### 错误 2：API Key 未配置

```
[请求 xxx] ❌ 错误: 用户未配置 Claude API Key
```

**原因：** 用户没有在 Web 界面配置 Claude API Key

**排查：**
1. 登录 Web 界面（http://localhost:8000）
2. 检查是否配置了 Claude API Key

**解决：**
1. 在 Web 界面配置 Claude API Key
2. 保存配置
3. 重新测试

### 错误 3：认证失败

```
[请求 xxx] 🔑 认证失败
[请求 xxx] 错误状态码: 401
```

**原因：** Claude API Key 无效或过期

**排查：**
1. 检查 Claude API Key 是否正确
2. 检查 API Key 是否有效
3. 检查账户余额

**解决：**
1. 更新 Claude API Key
2. 确认 API Key 有效
3. 充值账户

### 错误 4：模型不可用

```
[请求 xxx] 🚫 模型不可用
```

**原因：** 指定的模型不存在或不支持

**排查：**
1. 查看 `解析后参数` 中的模型名称
2. 检查是否拼写错误

**解决：**
使用支持的模型：
- claude-haiku-4-5-20251001
- claude-sonnet-4-5-20250929
- claude-opus-4-6

### 错误 5：请求超时

```
[请求 xxx] ⏰ 请求超时
```

**原因：** 
- 文本太长
- 模型太慢（Opus）
- 网络问题

**排查：**
1. 查看 `文本长度`
2. 查看使用的模型
3. 查看 API 调用耗时

**解决：**
1. 使用更快的模型（Haiku 或 Sonnet）
2. 减少文本长度
3. 检查网络连接

### 错误 6：连接失败

```
[请求 xxx] 🔌 连接失败
```

**原因：** 无法连接到 Claude API 服务器

**排查：**
1. 查看 `Claude API 地址`
2. 检查网络连接
3. 检查防火墙设置

**解决：**
1. 确认 API 地址正确
2. 检查网络连接
3. 尝试其他 API 地址

## 📋 排查步骤

### 步骤 1：查看完整日志

```bash
# 实时查看日志
pm2 logs seoapi --lines 100

# 或查看文件日志
tail -f /path/to/log/file
```

### 步骤 2：找到对应的请求 ID

在日志中搜索小旋风测试时的时间点，找到对应的请求 ID。

### 步骤 3：分析请求信息

查看该请求 ID 的所有日志，重点关注：

1. **请求参数是否正确**
   - text_length 是否大于 0
   - mode 和 model 是否正确
   - sim 参数是否符合预期

2. **认证是否成功**
   - 是否显示 "✓ Claude API Key 已配置"
   - 是否有认证失败的错误

3. **API 调用是否成功**
   - 是否显示 "✓ Claude API 调用成功"
   - 耗时是否正常（< 30秒）

4. **响应是否正确**
   - 是否显示 "✅ 响应已发送"
   - errcode 是否为 "0"

### 步骤 4：根据错误类型解决

根据日志中的错误标识（❌、🔑、🚫、⏰、🔌）找到对应的解决方案。


### 技巧 2：过滤特定请求 ID 的日志

```bash
pm2 logs seoapi | grep "\[请求 a1b2c3d4\]"
```

### 技巧 3：只看错误日志

```bash
pm2 logs seoapi | grep "❌"
```

```bash
# 统计成功的请求
pm2 logs seoapi | grep "✅ 响应已发送" | wc -l
```

```bash

```

## 📝 日志示例分析

### 示例 1：小旋风配置错误

**日志：**
```
[请求 abc123] 请求体参数: {"text_length":0,"mode":"default","model":"default","sim":undefined}
[请求 abc123] ❌ 错误: 文本为空
```

**分析：**
- text_length = 0：说明没有接收到文本
- mode = "default"：说明小旋风没有传递 mode 参数
- sim = undefined：说明小旋风没有传递 sim 参数

**结论：** 小旋风的 post 格式配置错误

**解决：**
```
post格式: txt={word}&mode=seo_fast&sim=1
```

### 示例 2：API Key 问题

**日志：**
```
[请求 def456] ✓ Claude API Key 已配置
[请求 def456] 🚀 开始调用 Claude API...
[请求 def456] ❌ 发生错误
[请求 def456] 错误状态码: 401
[请求 def456] 🔑 认证失败
```

**分析：**
- API Key 已配置但认证失败
- 错误状态码 401：未授权

**结论：** Claude API Key 无效或过期

**解决：** 更新 Claude API Key

### 示例 3：超时问题

**日志：**
```
[请求 ghi789] 解析后参数: 文本长度=3500, 模式=seo_fast, 模型=claude-opus-4-6
[请求 ghi789] 🚀 开始调用 Claude API...
[请求 ghi789] ⏰ 请求超时
```

**分析：**
- 文本长度 3500 字符
- 使用 Opus 模型（最慢）
- 请求超时

**结论：** Opus 模型处理长文本太慢

**解决：** 使用 Sonnet 或 Haiku 模型
