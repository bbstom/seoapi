# 数据库部署指南

## 快速开始

### 1. 安装 MySQL 8.4

#### Ubuntu/Debian
```bash
# 添加 MySQL APT 仓库
wget https://dev.mysql.com/get/mysql-apt-config_0.8.29-1_all.deb
sudo dpkg -i mysql-apt-config_0.8.29-1_all.deb
sudo apt update

# 安装 MySQL 8.4
sudo apt install mysql-server

# 启动 MySQL
sudo systemctl start mysql
sudo systemctl enable mysql

# 安全配置
sudo mysql_secure_installation
```

#### CentOS/RHEL
```bash
# 添加 MySQL Yum 仓库
sudo yum install https://dev.mysql.com/get/mysql84-community-release-el8-1.noarch.rpm

# 安装 MySQL 8.4
sudo yum install mysql-community-server

# 启动 MySQL
sudo systemctl start mysqld
sudo systemctl enable mysqld

# 获取临时密码
sudo grep 'temporary password' /var/log/mysqld.log

# 安全配置
sudo mysql_secure_installation
```

#### 宝塔面板
1. 登录宝塔面板
2. 软件商店 → 搜索 "MySQL"
3. 选择 MySQL 8.0+ 版本
4. 点击安装
5. 安装完成后，在"数据库"中创建数据库

### 2. 创建数据库和用户

```bash
# 登录 MySQL
mysql -u root -p

# 创建数据库
CREATE DATABASE seoapi CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

# 创建用户（替换密码）
CREATE USER 'seoapi'@'localhost' IDENTIFIED BY 'your_strong_password';

# 授权
GRANT ALL PRIVILEGES ON seoapi.* TO 'seoapi'@'localhost';
FLUSH PRIVILEGES;

# 退出
EXIT;
```

### 3. 配置项目

```bash
# 复制配置文件
cp .env.example .env

# 编辑配置文件
nano .env
```

配置内容：
```env
# 服务器配置
PORT=8000
NODE_ENV=production

# Claude API 配置（可选，用户可在 Web 界面配置）
CLAUDE_API_KEY=
CLAUDE_BASE_URL=https://api.api123.icu

# 数据库配置
DB_HOST=localhost
DB_PORT=3306
DB_USER=seoapi
DB_PASSWORD=your_strong_password
DB_NAME=seoapi
```

### 4. 安装依赖

```bash
npm install
```

### 5. 初始化数据库

```bash
# 方式 1：手动初始化（推荐）
node scripts/init-database.js

# 方式 2：启动时自动初始化
npm start
```

### 6. 迁移旧数据（如果有）

```bash
# 迁移用户数据
node scripts/migrate-users-to-db.js

# 迁移日志数据（可选）
node scripts/migrate-logs-to-db.js
```

### 7. 启动服务

```bash
# 开发模式
npm start

# 生产模式（使用 PM2）
npm install -g pm2
pm2 start server.js --name seoapi
pm2 save
pm2 startup
```

### 8. 验证部署

访问：`http://your-server:8000`

默认账号：
- 用户名：`admin`
- 密码：`admin123`

**⚠️ 重要：首次登录后请立即修改密码！**

Nginx 配置：
```nginx
location / {
    proxy_pass http://127.0.0.1:8000;
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection 'upgrade';
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
    proxy_cache_bypass $http_upgrade;
}
```

## 数据库维护

### 备份数据库

```bash
# 手动备份
mysqldump -u seoapi -p seoapi > backup_$(date +%Y%m%d).sql

# 定时备份（添加到 crontab）
0 2 * * * mysqldump -u seoapi -p'password' seoapi > /backup/seoapi_$(date +\%Y\%m\%d).sql
```

### 恢复数据库

```bash
mysql -u seoapi -p seoapi < backup_20260310.sql
```

### 清理旧日志

```bash
# 保留最近 90 天的日志
mysql -u seoapi -p seoapi -e "DELETE FROM api_logs WHERE created_at < DATE_SUB(NOW(), INTERVAL 90 DAY);"
```

### 优化数据库

```bash
# 优化表
mysql -u seoapi -p seoapi -e "OPTIMIZE TABLE api_logs, users, sessions;"

# 分析表
mysql -u seoapi -p seoapi -e "ANALYZE TABLE api_logs, users, sessions;"
```

## 性能优化

### 1. MySQL 配置优化

编辑 `/etc/mysql/my.cnf`：

```ini
[mysqld]
# 连接数
max_connections = 200

# 缓冲池大小（建议设置为物理内存的 50-80%）
innodb_buffer_pool_size = 1G

# 日志文件大小
innodb_log_file_size = 256M

# 查询缓存（MySQL 8.0 已移除，无需配置）

# 慢查询日志
slow_query_log = 1
slow_query_log_file = /var/log/mysql/slow.log
long_query_time = 2
```

重启 MySQL：
```bash
sudo systemctl restart mysql
```

### 2. 数据库索引

已自动创建的索引：
- `api_logs.username` - 用户查询
- `api_logs.status` - 状态筛选
- `api_logs.model` - 模型筛选
- `api_logs.mode` - 模式筛选
- `api_logs.created_at` - 日期排序
- `users.api_key` - API Key 验证
- `sessions.token` - 会话验证
- `sessions.expires_at` - 过期清理

## 监控和告警

### 1. 数据库监控

```bash
# 查看连接数
mysql -u seoapi -p -e "SHOW STATUS LIKE 'Threads_connected';"

# 查看慢查询
mysql -u seoapi -p -e "SHOW STATUS LIKE 'Slow_queries';"

# 查看表大小
mysql -u seoapi -p seoapi -e "
SELECT 
    table_name AS 'Table',
    ROUND(((data_length + index_length) / 1024 / 1024), 2) AS 'Size (MB)'
FROM information_schema.TABLES
WHERE table_schema = 'seoapi'
ORDER BY (data_length + index_length) DESC;
"
```

## 故障排查

### 问题 1：连接数据库失败

**错误信息：**
```
Error: connect ECONNREFUSED 127.0.0.1:3306
```

**解决方案：**
1. 检查 MySQL 是否启动：`sudo systemctl status mysql`
2. 检查端口是否监听：`netstat -tlnp | grep 3306`
3. 检查防火墙：`sudo ufw status`
4. 检查 `.env` 配置是否正确

### 问题 2：认证失败

**错误信息：**
```
Error: Access denied for user 'seoapi'@'localhost'
```

**解决方案：**
1. 检查用户名和密码是否正确
2. 重新授权：
```sql
GRANT ALL PRIVILEGES ON seoapi.* TO 'seoapi'@'localhost';
FLUSH PRIVILEGES;
```

### 问题 3：表不存在

**错误信息：**
```
Error: Table 'seoapi.users' doesn't exist
```

**解决方案：**
```bash
# 重新初始化数据库
node scripts/init-database.js
```

### 问题 4：性能慢

**解决方案：**
1. 检查慢查询日志
2. 优化数据库配置
3. 清理旧数据
4. 添加索引
