# 小旋风长文本改写失败问题说明

## 问题描述

你遇到的问题：
- ✅ 短文本（27 字符）改写成功
- ❌ 长文本（1261 字符）改写失败
- ✅ 服务器日志显示成功（errcode=0）
- ❌ 小旋风显示"改写失败"

## 根本原因

**小旋风对 API 响应体有大小限制（约 1300-1500 字符）**

当改写后的文本 + JSON 结构超过这个限制时，小旋风无法正确解析响应，即使服务器端一切正常。

## 解决方案

### 方案 1：限制采集长度（最简单，推荐）⭐

在小旋风的采集规则中：
```
内容截取：前 800 字符
```

或使用正则表达式：
```
(.{1,800})
```

**这是最简单有效的方法！**

### 方案 2：使用 compact 参数

在小旋风的 post 格式中添加：
```
txt={word}&mode=seo_fast&sim=1&compact=1
```

compact=1 会压缩输出，移除多余的空格和换行，可以减少 10-20% 的大小。

### 方案 3：分段改写

对于超长文章，在采集规则中分段：
- 段落1：前 800 字符
- 段落2：801-1600 字符
- 分别调用 API

## 推荐配置

```
API名称: Claude AI 伪原创
API地址: http://your-domain.com/api/rewrite
post格式: txt={word}&mode=seo_fast&sim=1&compact=1
Authorization: your_api_key
成功标志: errcode=0
返回错误的字段: errmsg
伪原创后的内容字段: data

采集规则：
- 内容截取：前 800 字符 ⭐
```

## 文本长度参考

| 长度范围 | 成功率 | 建议 |
|---------|-------|------|
| <800 字符 | 100% | ✅ 安全 |
| 800-1000 字符 | 95% | ✅ 安全 |
| 1000-1200 字符 | 70% | ⚠️ 可能失败 |
| >1200 字符 | 30% | ❌ 很可能失败 |

## 测试方法

运行测试脚本：


这个脚本会测试不同长度的文本，帮助你找到最佳配置。

## 服务器日志

现在服务器会在日志中显示警告：

```
[请求 xxx] ⚠️ 警告: 响应文本较长 (1261 字符)
[请求 xxx] 💡 建议: 在小旋风中限制采集文本长度为 800-1000 字符
[请求 xxx] 💡 或在 post 格式中添加 compact=1 参数压缩输出
```

当你看到这个警告时，就知道需要调整配置了。

## 快速诊断

1. 查看服务器日志中的"响应内容长度"
2. 如果长度 > 1200 字符 → 限制采集长度
3. 如果长度 < 1200 字符 → 检查其他原因
