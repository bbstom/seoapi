# 令牌负载均衡模型多选功能 - 部署指南

## 功能说明

为令牌的负载均衡模式添加了模型多选功能：

1. **自动加载模型列表**：选择负载均衡节点后，自动加载这些节点支持的所有模型
2. **模型多选**：支持从模型列表中选择多个允许使用的模型
3. **默认模型**：可以从允许的模型中选择一个作为默认模型
4. **数据持久化**：配置保存后刷新页面仍然保留

## 功能使用说明

### 负载均衡模式配置

1. **选择节点策略**：选择"负载均衡"
2. **选择负载均衡策略**：轮询、加权轮询、最少连接、随机
3. **选择参与节点**：勾选多个 API 节点（可选，不选则使用所有健康节点）
4. **选择允许的模型**：
   - 系统会自动加载所选节点支持的所有模型
   - 勾选允许使用的模型（可多选）
   - 如果不选，则允许使用所有模型
5. **选择默认模型**：从允许的模型中选择一个作为默认（可选）

### 数据库字段说明

- `allowed_models`：JSON 数组，存储允许使用的模型列表
  - 例如：`["gpt-4", "gpt-3.5-turbo", "claude-3"]`
  - 如果为 `NULL` 或空数组，表示允许所有模型
  
- `default_model`：字符串，存储默认模型
  - 例如：`"gpt-4"`
  - 如果设置了 `allowed_models`，`default_model` 必须在列表中

## 故障排查

### 问题 1：模型列表不显示
**原因**：所选节点没有可用模型
**解决**：
1. 进入"API 管理"页面
2. 找到对应的节点
3. 点击"测试连接"获取模型列表

### 问题 2：保存后刷新配置丢失
**原因**：前端未正确映射 `allowed_models` 字段
**解决**：已在本次修改中修复，确保前端已重新构建

### 问题 3：保存失败
**原因**：数据库字段不存在
**解决**：确保已执行数据库迁移脚本
