# 日志统计系统说明

## 功能概述

系统自动记录所有 API 调用的详细信息，包括：
- 调用次数
- 调用时间
- Token 使用情况（输入/输出）
- 使用的网站（Base URL）
- 使用的模型
- 改写模式
- 成功/失败状态
- 用户信息
- 客户端信息

## 日志存储

### 日志文件位置
```
logs/
├── api-2026-03-08.log    # 每日 API 调用日志
├── api-2026-03-09.log
├── stats.json            # 统计数据汇总
```

## 统计数据

### 统计维度

1. **总计统计**
   - 总调用次数
   - 成功次数
   - 失败次数
   - 总 Token 使用量
   - 总耗时

2. **按用户统计**
   - 每个用户的调用次数
   - 成功/失败次数
   - Token 使用量
   - 平均耗时

3. **按模型统计**
   - 每个模型的使用次数
   - 成功率
   - Token 使用量

4. **按模式统计**
   - 每个改写模式的使用次数
   - 成功率

5. **按 Base URL 统计**
   - 每个 API 服务的调用次数
   - 成功率

6. **按日期统计**
   - 每天的调用次数
   - Token 使用量
   - 成功率

## API 接口

### 1. 获取统计数据

**请求：**
```
GET /api/stats
Headers: x-session-token: your_session_token
```

**响应（普通用户）：**
```json
{
  "success": true,
  "stats": {
    "user": {
      "calls": 500,
      "success": 480,
      "error": 20,
      "totalTokens": 250000,
      "totalDuration": 1625.3
    },
    "lastUpdated": "2026-03-09T10:30:45.123Z"
  }
}
```

**响应（管理员）：**
```json
{
  "success": true,
  "stats": {
    "total": { ... },
    "byUser": { ... },
    "byModel": { ... },
    "byMode": { ... },
    "byBaseURL": { ... },
    "byDate": { ... },
    "lastUpdated": "2026-03-09T10:30:45.123Z"
  }
}
```

### 2. 获取用户日志

**请求：**
```
GET /api/logs?limit=100&username=admin
Headers: x-session-token: your_session_token
```

参数：
- `limit`: 返回的日志条数（默认 100）
- `username`: 用户名（仅管理员可指定，普通用户只能查看自己的）

**响应：**
```json
{
  "success": true,
  "logs": [
    {
      "timestamp": "2026-03-08T10:30:45.123Z",
      "requestId": "b2e3bc32fa347d3b",
      "username": "admin",
      "status": "success",
      "model": "claude-opus-4-6",
      "mode": "seo_fast",
      "duration": 3.25,
      "inputTokens": 245,
      "outputTokens": 189
    },
    ...
  ],
  "total": 100
}
```

### 3. 清理旧日志（仅管理员）

**请求：**
```
POST /api/logs/clean
Headers: x-session-token: your_session_token
Content-Type: application/json

{
  "daysToKeep": 30
}
```

**响应：**
```json
{
  "success": true,
  "message": "已清理 30 天前的日志"
}
```

## 日志管理

### 自动清理

系统不会自动清理日志，需要管理员手动清理：

```bash
# 通过 API 清理
curl -X POST http://localhost:8000/api/logs/clean \
  -H "x-session-token: your_token" \
  -H "Content-Type: application/json" \
  -d '{"daysToKeep": 30}'
```

### 手动清理

直接删除日志文件：

```bash
# 删除 30 天前的日志
find logs/ -name "api-*.log" -mtime +30 -delete
```

### 日志备份

建议定期备份日志：

```bash
# 备份到其他目录
tar -czf logs-backup-$(date +%Y%m%d).tar.gz logs/

# 上传到云存储
# aws s3 cp logs-backup-*.tar.gz s3://your-bucket/
```

## 性能考虑

### 1. 异步写入

日志写入是同步的，但非常快速（<1ms），不会影响 API 响应速度。

### 2. 文件大小

每条日志约 500 字节，1000 次调用约 500KB。

### 3. 磁盘空间

建议预留空间：
- 每天 10000 次调用 ≈ 5MB
- 保留 30 天 ≈ 150MB
- 保留 1 年 ≈ 1.8GB

### 4. 查询性能

- 统计数据查询：<10ms（内存读取）
- 日志查询：<100ms（文件读取）
- 大量日志查询：可能较慢，建议限制条数

## 隐私和安全

### 记录的信息

✅ 记录：
- 用户名
- 调用时间
- 模型和模式
- Token 使用量
- 客户端 IP
- User-Agent

❌ 不记录：
- API Key
- 原文内容
- 改写后的内容
- 密码

### 访问控制

- 普通用户：只能查看自己的统计和日志
- 管理员：可以查看所有统计和日志

### 数据保护

- 日志文件存储在服务器本地
- 不会发送到外部服务
- 建议定期备份和清理

## 常见问题

### Q1: 日志会占用多少磁盘空间？

A: 取决于调用频率：
- 低频（100次/天）：约 50KB/天
- 中频（1000次/天）：约 500KB/天
- 高频（10000次/天）：约 5MB/天

### Q2: 如何查看实时统计？

A: 访问 Web 界面的统计页面，或调用 `/api/stats` 接口。

### Q3: 日志会影响性能吗？

A: 不会。日志写入非常快速（<1ms），不会影响 API 响应。

### Q4: 可以导出日志吗？

A: 可以。日志文件是标准的 JSON Lines 格式，可以直接读取或导入到其他系统。

### Q5: 如何分析日志？

A: 可以使用：
- 命令行工具（grep, awk, jq）
- 日志分析工具（ELK, Splunk）
- 自定义脚本

## 示例分析

### 1. 统计每个用户的 Token 使用量

```bash
cat logs/api-2026-03-08.log | \
  jq -s 'group_by(.username) | map({username: .[0].username, totalTokens: map(.totalTokens) | add})' | \
  jq 'sort_by(.totalTokens) | reverse'
```

### 2. 查找失败的请求

```bash
cat logs/api-2026-03-08.log | \
  jq 'select(.status == "error")' | \
  jq -s 'group_by(.errorCode) | map({errorCode: .[0].errorCode, count: length})'
```

### 3. 计算平均响应时间

```bash
cat logs/api-2026-03-08.log | \
  jq -s 'map(.duration) | add / length'
```

### 4. 统计最常用的模型

```bash
cat logs/api-2026-03-08.log | \
  jq -s 'group_by(.model) | map({model: .[0].model, count: length}) | sort_by(.count) | reverse'
```
