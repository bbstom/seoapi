# 统计数据修复说明

## 🐛 问题描述

用户反馈 Web 界面显示的统计数据不正确：
- 耗时显示为 `-s`
- 相似度显示为 `30.2%`（实际应该是 `3.02%` 或 `0.0302`）
- 输入 Tokens 显示为 `-`
- 输出 Tokens 显示为 `-`

## 🔍 问题原因

服务器代码中，`_meta` 统计数据只在开发环境（`NODE_ENV === 'development'`）中返回：

```javascript
// 之前的代码
if (process.env.NODE_ENV === 'development') {
    response._meta = {
        mode: mode,
        model: model,
        duration: duration,
        usage: {
            input_tokens: message.usage.input_tokens,
            output_tokens: message.usage.output_tokens
        }
    };
}
```

由于没有设置 `NODE_ENV` 环境变量，系统默认不是开发环境，导致 `_meta` 数据不返回，前端无法获取统计信息。

## ✅ 解决方案

移除环境判断，让统计数据始终返回：

```javascript
// 修复后的代码
const response = {
    errcode: '0',
    errmsg: '',
    data: rewrittenText,
    like: similarity,
    _meta: {
        mode: mode,
        model: model,
        duration: duration,
        usage: {
            input_tokens: message.usage.input_tokens,
            output_tokens: message.usage.output_tokens
        }
    }
};
```

## 📊 修复效果

相似度: -%
输入 Tokens: -
输出 Tokens: -
```

### 修复后
```

## 📝 数据说明

### 1. 耗时（duration）

- **单位**：秒
- **含义**：从发送请求到收到响应的总时间
- **范围**：通常 2-20 秒，取决于模型和文本长度
- **示例**：7.13s

### 2. 相似度（like）

- **单位**：百分比
- **含义**：改写后文本与原文的相似度（使用 Jaccard 相似度算法）
- **范围**：0-100%，越低表示改写程度越大
- **计算方式**：字符级别的 Jaccard 相似度
- **示例**：1.9%（表示改写后与原文相似度很低，改写效果好）

**注意**：相似度算法比较简单，仅供参考。实际的语义相似度可能与此不同。

### 3. 输入 Tokens（input_tokens）

- **单位**：tokens
- **含义**：发送给 Claude API 的 tokens 数量（包括提示词和原文）
- **影响**：影响 API 调用成本
- **示例**：2427

**说明**：输入 tokens 包括：
- 改写模式的提示词（通常 100-500 tokens）
- 原文内容
- 系统提示词

### 4. 输出 Tokens（output_tokens）

- **单位**：tokens
- **含义**：Claude API 返回的 tokens 数量（改写后的文本）
- **影响**：影响 API 调用成本
- **示例**：180

## 💰 成本计算

根据 api123.icu 的定价（以 Claude Sonnet 4.5 为例）：

- **输入**：$10.0000/M tokens
- **输出**：$50.0000/M tokens

**示例计算：**
```
输入成本：2427 tokens × $10 / 1,000,000 = $0.02427
输出成本：180 tokens × $50 / 1,000,000 = $0.009
总成本：$0.03327（约 ¥0.24）
```

## 🎯 优化建议

### 1. 降低输入 Tokens

- 使用更简洁的提示词
- 对于长文本，考虑分段处理

### 2. 选择合适的模型

- **Haiku 4.5**：最便宜，适合大量简单改写
- **Sonnet 4.5**：性价比最高，推荐 ⭐
- **Opus 4.6**：质量最高，适合重要内容

### 3. 批量处理

如果有大量文本需要改写，建议：
- 使用更快的模型（Haiku）
- 设置合理的并发数
- 监控 API 使用量
