# 改写模式优化 - 说明文档

## 优化内容

1. 显示模式ID（蓝色背景标签）
2. 提示词详情改为可折叠（默认折叠）
3. 优化视觉层次和交互体验

## 修改内容

### 前端修改
**文件：** `frontend/src/pages/ModesPage.jsx`

1. **模式ID显示**
   - 蓝色背景（`bg-blue-50`）
   - 蓝色文字（`text-blue-700`）
   - 蓝色边框（`border-blue-200`）
   - 更加醒目，易于识别

2. **提示词折叠功能**
   - 添加状态管理 `expandedPrompts`
   - 默认折叠，节省页面空间
   - 点击展开/折叠
   - 显示箭头图标指示状态

3. **导入新图标**
   - `ChevronDown` - 向下箭头
   - `ChevronUp` - 向上箭头

## 显示效果对比

### 修改前
- 模式ID：灰色背景，不够醒目
- 提示词：始终展开，占用大量空间
- 多个模式时页面很长，需要大量滚动

### 修改后
- 模式ID：蓝色背景，清晰醒目
- 提示词：默认折叠，点击查看
- 页面更加紧凑，一屏可以看到更多模式

## 功能特性

### 1. 模式ID样式
```jsx
<code className="px-2.5 py-1 bg-blue-50 text-blue-700 text-xs font-mono rounded-md border border-blue-200">
  {mode.id}
</code>
```

### 2. 提示词折叠
- 默认状态：折叠
- 点击按钮：展开/折叠切换
- 独立状态：每个模式的展开状态互不影响
- 视觉反馈：箭头图标变化

### 3. 交互优化
- 折叠按钮悬停效果
- 平滑的展开/折叠动画
- 清晰的视觉层次

## 用途说明

### 模式ID的作用
1. **API调用**：`mode=seo_original`
2. **数据看板**：显示调用使用的模式
3. **配置文件**：在 `config/rewrite-modes.js` 中定义
4. **快速识别**：蓝色标签更容易找到对应模式

### 提示词折叠的好处
1. **节省空间**：一屏显示更多模式
2. **减少干扰**：只在需要时查看详情
3. **提升效率**：快速浏览模式列表
4. **保持整洁**：页面更加简洁

2. 进入"改写模式"页面
3. 检查模式ID：
   - 应该显示为蓝色背景标签
   - 文字清晰可读
4. 检查提示词：
   - 默认应该是折叠状态

5. 点击"提示词详情"：
   - 应该展开显示完整提示词
   - 箭头变为向上
6. 再次点击：
   - 应该折叠隐藏提示词
   - 箭头变为向下
7. 对比数据看板：
   - 蓝色模式ID与数据看板中的模式名称对应

## 技术实现

### 状态管理
```jsx
const [expandedPrompts, setExpandedPrompts] = useState({});

const togglePrompt = (modeId) => {
  setExpandedPrompts(prev => ({
    ...prev,
    [modeId]: !prev[modeId]
  }));
};
```

### 折叠组件
```jsx
<div className="border border-slate-200 rounded-xl overflow-hidden">
  {/* 折叠按钮 */}
  <button onClick={() => togglePrompt(mode.id)}>
    <span>提示词详情</span>
    {expandedPrompts[mode.id] ? <ChevronUp /> : <ChevronDown />}
  </button>
  
  {/* 提示词内容（条件渲染） */}
  {expandedPrompts[mode.id] && (
    <div className="p-4">
      <p>{mode.prompt}</p>
    </div>
  )}
</div>
```
