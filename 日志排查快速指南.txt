日志排查快速指南
========================================

现在每个请求都有详细日志，帮助排查问题！

========================================
🔍 查看日志
========================================

Windows 开发环境：
直接在 PowerShell/CMD 窗口查看
node server.js

或使用 PM2：
pm2 start server.js --name seoapi
pm2 logs seoapi --lines 100

Linux 生产环境：
pm2 logs seoapi --lines 100

查看错误：
pm2 logs seoapi | grep "❌"

查看最近 50 行：
pm2 logs seoapi --lines 50

========================================
🪟 Windows 特别说明
========================================

方法 1: 直接查看（推荐）
cd seoapi-nodejs
node server.js
# 日志会实时显示在控制台

方法 2: 保存到文件
node server.js > logs.txt 2>&1
# 在另一个窗口查看
Get-Content logs.txt -Wait -Tail 50

方法 3: 使用 VS Code
1. 在 VS Code 中打开项目
2. 按 Ctrl+` 打开终端
3. 运行 node server.js

方法 4: 使用 PM2
pm2 start server.js --name seoapi
pm2 logs seoapi

详细说明：查看 Windows日志查看指南.md

========================================
📋 日志格式
========================================

每个请求都有唯一 ID：
[请求 a1b2c3d4e5f6g7h8] ...

日志包含：
1. 请求信息（用户、IP、User-Agent）
2. 请求参数（文本长度、模式、模型）
3. 处理过程（验证、API 调用、耗时）
4. 响应信息（输出长度、格式、状态）
5. 错误信息（类型、消息、详情）

========================================
🎯 日志标识
========================================

✓  成功
✅ 完成
❌ 错误
⚠️  警告
🚀 开始
📤 发送
🔑 认证失败
🚫 模型不可用
⏰ 超时
🔌 连接失败

========================================
🔍 常见错误排查
========================================

错误 1: 文本为空
日志: [请求 xxx] ❌ 错误: 文本为空
原因: 小旋风配置错误
解决: post格式: txt={word}&mode=seo_fast&sim=1

错误 2: API Key 未配置
日志: [请求 xxx] ❌ 错误: 用户未配置 Claude API Key
原因: 未配置 Claude API Key
解决: 在 Web 界面配置

错误 3: 认证失败
日志: [请求 xxx] 🔑 认证失败
原因: API Key 无效
解决: 更新 Claude API Key

错误 4: 模型不可用
日志: [请求 xxx] 🚫 模型不可用
原因: 模型名称错误
解决: 使用支持的模型

错误 5: 请求超时
日志: [请求 xxx] ⏰ 请求超时
原因: 文本太长或模型太慢
解决: 使用 Haiku 或 Sonnet 模型

错误 6: 连接失败
日志: [请求 xxx] 🔌 连接失败
原因: 无法连接 API 服务器
解决: 检查网络和 API 地址

========================================
📊 排查步骤
========================================

步骤 1: 在小旋风中测试
点击"测试"按钮

步骤 2: 查看服务器日志
pm2 logs seoapi --lines 100

步骤 3: 找到对应的请求
查找测试时间点的日志
记下请求 ID

步骤 4: 分析请求流程
查看该请求 ID 的所有日志
重点关注：
- 请求参数是否正确
- 是否有错误标识
- 错误发生在哪个环节

步骤 5: 根据错误解决
参考上面的常见错误
或查看详细日志说明.md

========================================
💡 使用技巧
========================================

技巧 1: 过滤特定用户
pm2 logs seoapi | grep "\[admin\]"

技巧 2: 过滤特定请求
pm2 logs seoapi | grep "\[请求 a1b2c3d4\]"

技巧 3: 只看错误
pm2 logs seoapi | grep "❌"

技巧 4: 统计成功率
成功: pm2 logs seoapi | grep "✅ 响应已发送" | wc -l
失败: pm2 logs seoapi | grep "❌ 错误响应已发送" | wc -l

技巧 5: 查看最近错误
pm2 logs seoapi --err --lines 50
