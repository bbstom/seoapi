# 默认配置说明

## 🎯 功能说明

现在你可以在 Web 后台为每个用户设置**默认的模型和改写模式**。

当小旋风调用 API 时：
- 如果**指定了**模型/模式参数 → 使用指定的值
- 如果**没有指定**模型/模式参数 → 使用用户配置的默认值

## 📝 配置步骤

### 1. 登录系统

访问 `http://localhost:8000`，使用 admin/admin123 登录

### 2. 进入配置页面

点击顶部的"配置"标签页

### 3. 设置默认配置

在"默认调用配置"区域：

**默认模型**：
- Claude Haiku 4.5 - 最快速
- Claude Sonnet 4.5 - 推荐 ⭐（默认）
- Claude Sonnet 4.5 Thinking - 深入分析
- Claude Sonnet 4.6 - 最新版
- Claude Opus 4.5 - 强大
- Claude Opus 4.5 Thinking - 最强分析
- Claude Opus 4.6 - 顶级

**默认改写模式**：
- standard - 标准改写
- humanlike - 仿人类写作 ⭐（默认）
- mixed - 混合改写 ⭐
- deep - 深度重写 ⭐
- creative - 创意改写
- formal - 正式改写
- simple - 简化改写
- translate_cn - 翻译改写

### 4. 保存配置

点击"保存默认配置"按钮

## 🔄 使用场景

### 场景 1: 使用默认配置（简化小旋风配置）

**小旋风配置**：
```
post格式: txt={word}&sim=1
```

系统会自动使用你配置的默认模型和模式（如：claude-sonnet-4-5-20250929 + humanlike）

### 场景 2: 覆盖默认配置（灵活调整）

**小旋风配置**：
```
post格式: txt={word}&mode=deep&model=claude-opus-4-6&sim=1
```

系统会使用你指定的模型和模式，忽略默认配置

### 场景 3: 部分覆盖

**只指定模式**：
```
post格式: txt={word}&mode=creative&sim=1
```
使用指定的 creative 模式 + 默认模型

**只指定模型**：
```
post格式: txt={word}&model=claude-haiku-4-5-20251001&sim=1
```
使用指定的 Haiku 模型 + 默认模式

## 🎨 多用户场景

### 场景：团队使用

管理员可以为不同用户设置不同的默认配置：

**用户 A（新闻采集）**：
- 默认模型：Haiku 4.5
- 默认模式：standard

**用户 B（文章采集）**：
- 默认模型：Sonnet 4.5
- 默认模式：humanlike

**用户 C（精品采集）**：
- 默认模型：Opus 4.6
- 默认模式：deep

每个用户使用自己的 API Key，系统会自动使用对应的默认配置。

## 📊 配置优先级

```
指定的参数 > 用户默认配置 > 系统默认值
```

**示例**：

1. **完全使用默认**
   ```
   txt={word}&sim=1
   ```
   → 使用用户配置的默认模型和模式

2. **指定模式**
   ```
   txt={word}&mode=creative&sim=1
   ```
   → 使用 creative 模式 + 用户默认模型

3. **指定模型**
   ```
   txt={word}&model=claude-opus-4-6&sim=1
   ```
   → 使用 Opus 4.6 模型 + 用户默认模式

4. **全部指定**
   ```
   txt={word}&mode=deep&model=claude-opus-4-6&sim=1
   ```
   → 使用指定的模式和模型，忽略默认配置
