# 令牌节点配置功能完成 ✅

## 功能概述

为 API 令牌添加了节点配置功能，支持为每个令牌单独配置使用固定 API 节点或负载均衡模式。

## 新增功能

### 1. 数据库字段

在 `api_keys` 表中新增两个字段：

- `node_strategy` - 节点选择策略
  - `load_balance`: 负载均衡（默认）
  - `fixed`: 固定节点
  
- `fixed_node_id` - 固定节点ID
  - 当策略为 `fixed` 时，指定使用的 API 配置节点

### 2. 令牌管理界面

在"令牌管理"页面，每个令牌卡片新增"节点配置"区域：

#### 显示模式
- 显示当前节点策略（负载均衡/固定节点）
- 如果是固定节点，显示节点名称
- 点击"配置"按钮进入编辑模式

#### 编辑模式
- 节点策略选择：负载均衡 / 固定节点
- 固定节点选择：从可用的 API 配置中选择
- 保存/取消按钮

### 3. API 调用逻辑

改写请求的节点选择优先级：

1. **令牌固定节点**（最高优先级）
   - 如果令牌配置为 `fixed` 模式且指定了节点ID
   - 直接使用该固定节点
   - 如果节点不存在或已禁用，降级到负载均衡

2. **用户默认配置**
   - 使用用户在"个人中心"设置的默认 API 配置

3. **标记为默认的配置**
   - 使用标记为默认的 API 配置

4. **个人中心旧配置**（兼容性）
   - 降级到个人中心的 API Key 配置

## 使用场景

### 场景 1: 不同令牌使用不同节点

```
令牌A（小旋风专用）→ 固定节点：iflow API
令牌B（测试专用）   → 固定节点：测试 API
令牌C（通用）       → 负载均衡
```

### 场景 2: 高优先级任务使用专用节点

```
VIP令牌 → 固定节点：高速专线节点
普通令牌 → 负载均衡（自动选择最优节点）
```

### 场景 3: 按应用分配节点

```
应用A令牌 → 固定节点：节点1
应用B令牌 → 固定节点：节点2
应用C令牌 → 负载均衡
```

### 4. 改写逻辑集成

在 `/api/rewrite` 端点中：

```javascript
// 检查令牌的节点配置策略
if (req.user.nodeStrategy === 'fixed' && req.user.fixedNodeId) {
    // 使用固定节点
    apiConfig = await configManager.getConfig(req.user.username, req.user.fixedNodeId);
} else {
    // 使用负载均衡或默认配置
}
```

## 配置步骤

### 1. 运行数据库迁移

```bash
cd seoapi-nodejs
node scripts/add-token-node-config.js
```

### 2. 重启服务

```bash
# 停止现有服务
pm2 stop seoapi

# 启动服务
pm2 start ecosystem.config.js
```

### 3. 配置令牌节点

1. 登录系统
2. 进入"令牌管理"页面
3. 找到要配置的令牌
4. 点击"配置"按钮
5. 选择节点策略：
   - 负载均衡：自动选择最优节点
   - 固定节点：选择指定的 API 节点
6. 点击"保存"

## 优势

1. **灵活性**
   - 每个令牌可以独立配置节点策略
   - 支持混合使用固定节点和负载均衡

2. **可靠性**
   - 固定节点不可用时自动降级
   - 不影响现有令牌的正常使用

3. **可追踪性**
   - 日志清晰记录节点选择过程
   - 便于问题排查和性能分析

4. **向后兼容**
   - 默认使用负载均衡模式
   - 不影响现有系统功能

5. **精确控制**
   - 令牌固定节点时禁用故障转移
   - 确保使用指定的节点，不会自动切换

## 注意事项

1. **固定节点必须启用**
   - 如果固定节点被禁用，会自动降级到负载均衡
   - 建议定期检查固定节点状态

2. **节点配置优先级**
   - 令牌固定节点 > 用户默认配置 > 系统默认配置
   - 确保理解优先级顺序

3. **性能考虑**
   - 固定节点可能导致负载不均
   - 建议根据实际情况合理分配

4. **故障转移策略**
   - 令牌固定节点时：禁用故障转移，失败直接返回错误
   - 负载均衡模式时：启用故障转移，自动切换到其他节点
   - 这确保了用户的精确控制需求
