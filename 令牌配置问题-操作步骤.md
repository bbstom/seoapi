# 令牌配置保存问题 - 操作步骤

## 问题现象
保存令牌的负载均衡配置后，刷新页面配置丢失

## 已完成的修复
1. ✅ 前端添加 `allowed_models` 字段映射
2. ✅ 后端路由添加 `allowedModels` 参数接收
3. ✅ 添加调试日志查看数据流

## 需要执行的步骤

### 步骤 1：重启后端服务
```bash
cd /www/wwwroot/api.vpno.eu.org/seoapi
pm2 restart seoapi
```

### 步骤 2：查看当前数据库状态
```bash
cd /www/wwwroot/api.vpno.eu.org/seoapi
node scripts/check-token-data.js
```

### 步骤 3：测试保存配置
1. 打开浏览器访问 https://api.vpno.eu.org/
2. 登录后进入"令牌管理"
3. 点击某个令牌的"配置节点"
4. 选择"负载均衡"
5. 选择负载均衡策略（如：轮询）
6. 勾选 1-2 个 API 节点
7. 如果有模型列表，勾选 1-2 个模型
8. 点击"保存"

### 步骤 4：查看后端日志
```bash
pm2 logs seoapi --lines 30
```

**查找以下日志：**
- `[令牌配置] 接收到的数据:` - 查看前端发送的数据格式
- `[令牌配置] 准备保存的配置:` - 查看准备保存到数据库的数据
- `用户 xxx 更新令牌节点配置` - 确认保存成功

### 步骤 5：再次检查数据库
```bash
node scripts/check-token-data.js
```

查看数据是否正确保存

### 步骤 6：刷新页面验证
1. 刷新浏览器页面
2. 再次点击"配置节点"
3. 查看配置是否保留

## 预期结果

### 日志应该显示：
```
[令牌配置] 接收到的数据: {
  nodeStrategy: 'load_balance',
  loadBalanceStrategy: 'round_robin',
  loadBalanceNodes: [1, 2],  // 应该是数组

  allowedModels: ['gpt-4', 'gpt-3.5-turbo'],  // 应该是数组
  allowedModelsType: 'object',
  defaultModel: 'gpt-4'
}
```

### 数据库应该显示：
```
令牌 ID: 1
  负载均衡节点 (原始): [1,2]
  负载均衡节点 (解析): [1,2]
  允许的模型 (原始): ["gpt-4","gpt-3.5-turbo"]
  允许的模型 (解析): ["gpt-4","gpt-3.5-turbo"]
```

## 如果仍然有问题

### 问题 1：日志显示数据类型不对
如果日志显示 `loadBalanceNodesType: 'string'`，说明前端发送的数据格式有问题。

**解决方法：**
需要检查前端代码，确保发送的是数组而不是字符串。

### 问题 2：数据库中是字符串而不是 JSON
如果数据库显示 `"1,2"` 而不是 `[1,2]`，说明保存时没有正确序列化。

**解决方法：**
检查 `tokenManager.js` 的 `updateTokenNodeConfig` 方法。

### 问题 3：解析失败
如果看到 `解析 xxx 失败` 的错误，说明数据库中有损坏的数据。

**解决方法：**
```bash
# 清理损坏的数据
node scripts/fix-json-fields.js
```
