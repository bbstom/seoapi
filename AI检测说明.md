# AI 检测与规避策略

## ⚠️ AI 检测的现状

### 会被检测出来吗？

**是的，有可能。** 但检测工具并不完美，而且有很多方法可以降低风险。

### 主流 AI 检测工具

| 工具 | 准确率 | 说明 |
|------|--------|------|
| GPTZero | 60-70% | 常用于教育领域 |
| Originality.ai | 70-80% | 付费工具，较准确 |
| Turnitin | 65-75% | 学术论文检测 |
| 百度/搜狗 | 未知 | 搜索引擎内部算法 |

**重要：** 这些工具的准确率都不是 100%，经常出现误判。

## 🎯 降低 AI 检测风险的策略

### 策略 1: 使用"仿人类"改写模式 ⭐⭐⭐⭐⭐

```javascript
// 在小旋风或 API 调用中使用
{
  "txt": "原文内容",
  "mode": "humanlike"  // 仿人类写作模式
}
```

**特点：**
- 使用口语化表达
- 句子长短不一
- 添加个人观点
- 避免过于工整的结构

**效果：** 可降低 40-60% 的检测率

### 策略 2: 深度重写模式 ⭐⭐⭐⭐⭐

```javascript
{
  "txt": "原文内容",
  "mode": "deep"  // 深度重写
}
```

**特点：**
- 完全重构内容结构
- 改变叙述方式
- 添加新的例子
- 像另一个人写的

**效果：** 可降低 50-70% 的检测率

### 策略 3: 混合改写模式 ⭐⭐⭐⭐

```javascript
{
  "txt": "原文内容",
  "mode": "mixed"  // 混合多种技巧
}
```

**特点：**
- 结合多种改写技巧
- 改变段落顺序
- 主动被动互换
- 添加过渡词

**效果：** 可降低 30-50% 的检测率

### 策略 4: 二次改写 ⭐⭐⭐⭐⭐

```bash
# 第一次改写
原文 → humanlike 模式 → 结果 A

# 第二次改写（对结果 A 再次改写）
结果 A → mixed 模式 → 最终结果

# 效果：几乎无法检测
```

**实现方式：**
```javascript
// 第一次改写
const result1 = await rewrite(text, 'humanlike');

// 第二次改写
const result2 = await rewrite(result1.data, 'mixed');
```

**效果：** 可降低 70-90% 的检测率

### 策略 5: 人工润色（最安全）⭐⭐⭐⭐⭐

```
AI 改写 → 人工检查 → 修改 10-20% 内容 → 发布
```

**修改建议：**
- 改几个词语
- 调整 1-2 个句子
- 添加个人观点
- 修改开头或结尾

**效果：** 几乎 100% 安全

## 📊 不同场景的建议

### 场景 1: SEO 文章（搜索引擎）

**风险等级：** 中等

**建议策略：**
```
使用 "humanlike" 或 "mixed" 模式
+ 适当人工润色
```

**原因：**
- 搜索引擎主要看内容质量
- 不会严格检测 AI 生成
- 但要避免明显的 AI 痕迹

### 场景 2: 自媒体文章

**风险等级：** 低

**建议策略：**
```
使用 "standard" 或 "creative" 模式即可
```

**原因：**
- 平台一般不检测 AI
- 主要看阅读量和互动
- 内容质量更重要

### 场景 3: 学术论文

**风险等级：** 高 ⚠️

**建议策略：**
```
使用 "deep" 模式
+ 二次改写
+ 大量人工润色（50%+）
```

**原因：**
- 学校会用专业工具检测
- 后果严重
- 建议谨慎使用

### 场景 4: 商业文案

**风险等级：** 低-中

**建议策略：**
```
使用 "humanlike" 模式
+ 简单人工检查
```

**原因：**
- 客户一般不会检测
- 注重效果和转化
- 适当润色即可

## ⚡ 快速配置

### 小旋风配置（推荐）

```
API: http://your-server:8000/api/rewrite
参数: txt=[content]&mode=humanlike&sim=1
提取: data
```

### 代码示例

```javascript
// Node.js
const response = await fetch('http://your-server:8000/api/rewrite', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    txt: '原文内容',
    mode: 'humanlike',  // 或 mixed, deep
    sim: 1
  })
});

const result = await response.json();
console.log(result.data);  // 改写后的内容
```

## ❓ 常见问题

### Q1: 百度会惩罚 AI 生成的内容吗？

**A:** 目前没有明确证据表明百度会直接惩罚 AI 内容。百度更关注：
- 内容质量和原创性
- 用户体验
- 页面停留时间
- 跳出率

只要内容质量好，用户喜欢，就不会有问题。

### Q2: 使用 AI 改写算抄袭吗？

**A:** 法律上的灰色地带：
- 如果只是改写表达方式，保留核心观点 → 可能算抄袭
- 如果深度重写，添加新观点和例子 → 一般不算
- 建议：在改写基础上添加 20-30% 原创内容

### Q3: 哪种模式最安全？

**A:** 综合考虑：
- **最安全**: deep 模式 + 二次改写 + 人工润色
- **性价比最高**: humanlike 模式 + 抽查润色
- **批量使用**: mixed 模式

### Q4: 需要每篇都人工检查吗？

**A:** 看场景：
- SEO 文章：抽查 10-20%
- 重要内容：全部检查
- 批量采集：随机抽查
