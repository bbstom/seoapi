# 生产环境部署指南

## 📋 系统要求

### 服务器配置
- **操作系统**: Linux (推荐 Ubuntu 20.04+ / CentOS 7+)
- **CPU**: 2核心以上
- **内存**: 4GB以上
- **硬盘**: 20GB以上
- **Node.js**: 18.0.0 或更高版本（推荐 22.x LTS）
- **MySQL**: 8.0 或更高版本
- **Nginx**: 1.18 或更高版本（可选，用于反向代理）

### 网络要求
- 开放端口 8000（或自定义端口）
- 如果使用 Nginx，开放端口 80 和 443

## 🚀 快速部署（5步完成）

### 第1步：安装依赖

```bash
# 更新系统
sudo apt update && sudo apt upgrade -y

# 安装 Node.js 22.x LTS
curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
sudo apt install -y nodejs

# 验证安装
node --version  # 应显示 v22.x.x
npm --version   # 应显示 10.x.x

# 安装 MySQL 8.0
sudo apt install -y mysql-server

# 安装 PM2（进程管理器）
sudo npm install -g pm2

# 安装 Nginx（可选）
sudo apt install -y nginx
```

### 第2步：上传代码

```bash
# 创建项目目录
sudo mkdir -p /var/www/seoapi
sudo chown -R $USER:$USER /var/www/seoapi

# 上传代码（使用 git 或 scp）
cd /var/www/seoapi
git clone <your-repo-url> .

# 或使用 scp 上传
# scp -r ./seoapi-nodejs user@server:/var/www/seoapi
```

### 第3步：配置环境

```bash
# 进入项目目录
cd /var/www/seoapi

# 安装依赖
npm install --production

# 复制环境变量配置
cp .env.example .env

# 编辑环境变量
nano .env
```

**环境变量配置：**

```env
# 服务器配置
PORT=8000
NODE_ENV=production

# 数据库配置
DB_HOST=localhost
DB_USER=seoapi_user
DB_PASSWORD=your_secure_password_here
DB_NAME=seoapi
DB_PORT=3306

# 会话密钥（必须修改为随机字符串）
SESSION_SECRET=your_random_secret_key_here

# 日志配置
LOG_LEVEL=info
```

### 第4步：初始化数据库

```bash
# 登录 MySQL
sudo mysql -u root -p

# 创建数据库和用户
CREATE DATABASE seoapi CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'seoapi_user'@'localhost' IDENTIFIED BY 'your_secure_password_here';
GRANT ALL PRIVILEGES ON seoapi.* TO 'seoapi_user'@'localhost';
FLUSH PRIVILEGES;
EXIT;

# 运行初始化脚本
node scripts/init-database.js
```

## 🔧 配置 Nginx 反向代理（推荐）

### 创建 Nginx 配置

```bash
sudo nano /etc/nginx/sites-available/seoapi
```

**配置内容：**

```nginx
server {
    listen 80;
    server_name your-domain.com;  # 修改为你的域名

    # 日志
    access_log /var/log/nginx/seoapi_access.log;
    error_log /var/log/nginx/seoapi_error.log;

    # 客户端最大请求体大小
    client_max_body_size 10M;

    # 反向代理到 Node.js
    location / {
        proxy_pass http://localhost:8000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
        
        # 超时设置
        proxy_connect_timeout 300s;
        proxy_send_timeout 300s;
        proxy_read_timeout 300s;
    }
}
```

### 启用配置

```bash
# 创建软链接
sudo ln -s /etc/nginx/sites-available/seoapi /etc/nginx/sites-enabled/

# 测试配置
sudo nginx -t

# 重启 Nginx
sudo systemctl restart nginx
```

### 配置 HTTPS（可选但推荐）

```bash
# 安装 Certbot
sudo apt install -y certbot python3-certbot-nginx

# 获取 SSL 证书
sudo certbot --nginx -d your-domain.com

# 自动续期
sudo certbot renew --dry-run
```

## 🔒 安全加固

### 1. 修改默认密码

首次登录后立即修改管理员密码：
1. 访问 `http://your-domain.com`
2. 使用 `admin/admin123` 登录
3. 进入"个人中心"修改密码

### 2. 配置防火墙

```bash
# 安装 UFW
sudo apt install -y ufw

# 允许必要端口
sudo ufw allow 22/tcp    # SSH
sudo ufw allow 80/tcp    # HTTP
sudo ufw allow 443/tcp   # HTTPS

# 启用防火墙
sudo ufw enable

# 查看状态
sudo ufw status
```

### 3. 限制数据库访问

```bash
# 编辑 MySQL 配置
sudo nano /etc/mysql/mysql.conf.d/mysqld.cnf

# 确保只监听本地
bind-address = 127.0.0.1

# 重启 MySQL
sudo systemctl restart mysql
```

### 4. 设置文件权限

```bash
cd /var/www/seoapi

# 设置目录权限
sudo chown -R $USER:$USER .
chmod 755 .

# 保护敏感文件
chmod 600 .env
chmod 600 data/*.json

# 日志目录权限
chmod 755 logs
```

### 5. 配置速率限制

在 `.env` 中配置：

```env
# 速率限制（生产环境）
RATE_LIMIT_WINDOW=15  # 分钟
RATE_LIMIT_MAX=5      # 最大登录尝试次数
```

### 数据库备份

```bash
# 创建备份脚本
nano /var/www/seoapi/scripts/backup-database.sh
```

**备份脚本内容：**

```bash
#!/bin/bash

# 配置
DB_NAME="seoapi"
DB_USER="seoapi_user"
DB_PASS="your_password"
BACKUP_DIR="/var/backups/seoapi"
DATE=$(date +%Y%m%d_%H%M%S)

# 创建备份目录
mkdir -p $BACKUP_DIR

# 备份数据库
mysqldump -u $DB_USER -p$DB_PASS $DB_NAME | gzip > $BACKUP_DIR/seoapi_$DATE.sql.gz

# 删除30天前的备份
find $BACKUP_DIR -name "seoapi_*.sql.gz" -mtime +30 -delete

echo "备份完成: seoapi_$DATE.sql.gz"
```

```bash
# 设置执行权限
chmod +x /var/www/seoapi/scripts/backup-database.sh

# 添加到 crontab（每天凌晨2点备份）
crontab -e
# 添加以下行：
0 2 * * * /var/www/seoapi/scripts/backup-database.sh
```

### 日志管理

```bash
# 配置日志轮转
sudo nano /etc/logrotate.d/seoapi
```

**日志轮转配置：**

```
/var/www/seoapi/logs/*.log {
    daily
    rotate 30
    compress
    delaycompress
    notifempty
    create 0644 www-data www-data
    sharedscripts
    postrotate
        pm2 reloadLogs
    endscript
}
```

## 🎯 性能优化建议

### Node.js 优化

```bash
# 在 PM2 配置中设置
pm2 start server.js --name seoapi -i max  # 使用所有 CPU 核心
```

### MySQL 优化

```bash
# 编辑 MySQL 配置
sudo nano /etc/mysql/mysql.conf.d/mysqld.cnf

# 添加优化配置
[mysqld]
max_connections = 200
innodb_buffer_pool_size = 1G
innodb_log_file_size = 256M
query_cache_size = 64M
```

### Nginx 优化

```nginx
# 在 nginx.conf 中添加
gzip on;
gzip_vary on;
gzip_min_length 1024;
gzip_types text/plain text/css text/xml text/javascript application/json application/javascript;

# 缓存静态文件
location ~* \.(jpg|jpeg|png|gif|ico|css|js)$ {
    expires 1y;
    add_header Cache-Control "public, immutable";
}
```
