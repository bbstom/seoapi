# 小旋风改写失败排查指南

## 🔍 问题现象

**症状：**
- 后台日志显示改写成功
- 小旋风提示"改写失败"
- 部分请求成功，部分请求失败

**日志示例：**

```
✅ 成功的请求：
[admin] 改写完成 - 耗时: 4.08秒, 输出长度: 48
小旋风显示：改写成功

❌ 失败的请求：
[admin] 改写完成 - 耗时: 16.69秒, 输出长度: 1260
小旋风显示：改写失败
```

## 🎯 可能的原因

### 1. 响应格式问题

**原因：** 小旋风可能不支持 `_meta` 字段

**解决方案：** 已修改为只在开发环境返回 `_meta`

```javascript
// 生产环境返回纯净格式
{
  "errcode": "0",
  "errmsg": "",
  "data": "改写后的文本",
  "like": "0.5521"
}

// 开发环境或 debug=1 时返回完整格式
{
  "errcode": "0",
  "errmsg": "",
  "data": "改写后的文本",
  "like": "0.5521",
  "_meta": {
    "mode": "seo_fast",
    "model": "claude-opus-4-6",
    "duration": 16.69,
    "usage": {...}
  }
}
```

### 2. 响应内容包含特殊字符

**原因：** 改写后的文本可能包含：
- BOM 字符（\uFEFF）
- 不可见的控制字符
- 特殊的 Unicode 字符

**解决方案：** 已添加文本清理

```javascript
// 移除 BOM 和其他不可见字符
rewrittenText = rewrittenText.replace(/^\uFEFF/, '').trim();
```

### 3. 响应体太大

**原因：** 小旋风可能对响应大小有限制

**分析：**
- 失败的请求输出长度：1260、1029 字符
- 成功的请求输出长度：48 字符

**解决方案：** 
- 使用更简洁的响应格式（已实现）
- 移除 `_meta` 字段减小响应体积

### 4. 响应超时

**原因：** 小旋风可能有超时限制

**分析：**
- 失败的请求耗时：16.69秒、14.50秒
- 成功的请求耗时：4.08秒

**解决方案：** 使用更快的模型

```
推荐配置：
- 短文本（<500字）：claude-opus-4-6
- 中等文本（500-2000字）：claude-sonnet-4-5-20250929
- 长文本（>2000字）：claude-haiku-4-5-20251001
```

### 5. 小旋风解析错误

**原因：** 小旋风可能对 JSON 格式有严格要求

**检查项：**
- ✅ `errcode` 必须是字符串 `"0"`
- ✅ `errmsg` 必须存在（可以为空字符串）
- ✅ `data` 必须存在且包含改写后的文本
- ✅ 不能有额外的未知字段（如 `_meta`）

## ✅ 已实施的修复

### 修复 1：简化响应格式

```javascript
// 默认返回纯净格式（兼容小旋风）
const response = {
    errcode: '0',
    errmsg: '',
    data: rewrittenText,
    like: similarity
};

// 只在需要时返回元数据
if (process.env.NODE_ENV === 'development' || req.query.debug === '1') {
    response._meta = {...};
}
```

### 修复 2：清理响应文本

```javascript
// 移除 BOM 和不可见字符
rewrittenText = rewrittenText.replace(/^\uFEFF/, '').trim();
```

### 修复 3：优化日志输出

```javascript
// 避免日志过长导致问题
console.log(`[${req.user.username}] 返回响应:`, JSON.stringify(response).substring(0, 200));
```

## 📋 故障排查清单

当小旋风提示改写失败时，按以下步骤检查：

### 1. 检查服务器日志

```bash
pm2 logs seoapi --lines 50
```

**查看：**
- [ ] 是否显示"改写完成"
- [ ] 输出长度是多少
- [ ] 耗时是多少秒
- [ ] 是否有错误信息

### 2. 检查响应格式

```bash
# 使用 curl 测试
curl -X POST https://api.vpno.eu.org/api/rewrite \
  -H "Authorization: your_api_key" \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "txt=测试文本&mode=seo_fast&sim=1" \
  -v
```

**检查：**
- [ ] HTTP 状态码是否为 200
- [ ] Content-Type 是否为 application/json
- [ ] 响应体是否完整
- [ ] errcode 是否为 "0"

### 3. 检查文本长度和模型

**规则：**
- [ ] 文本 < 500 字 → 可用任何模型
- [ ] 文本 500-2000 字 → 推荐 Sonnet
- [ ] 文本 > 2000 字 → 推荐 Haiku

### 4. 检查小旋风配置

**确认：**
- [ ] API 地址正确
- [ ] post 格式正确
- [ ] Authorization 正确
- [ ] 成功标志为 `errcode=0`（不带引号）
- [ ] 返回字段为 `data`

### 5. 测试不同场景

```bash
# 测试 1：短文本 + Haiku
txt=测试&model=claude-haiku-4-5-20251001

# 测试 2：中等文本 + Sonnet
txt=中等长度的测试文本...&model=claude-sonnet-4-5-20250929

# 测试 3：不指定模型（使用默认）
txt=测试文本
```
