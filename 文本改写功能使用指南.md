# 文本改写功能使用指南

## 📖 概述

本系统提供两种文本改写调用方式：

1. **Web 界面改写** - 通过浏览器可视化界面进行改写
2. **API 接口调用** - 通过 HTTP API 进行程序化调用（适用于小旋风等第三方工具）

---

## 🌐 方式一：Web 界面改写

### 访问方式

1. 打开浏览器访问：`http://localhost:8000`（或你的服务器地址）
2. 使用账号密码登录系统
3. 默认进入"文本改写"页面

### 使用步骤

#### 1. 输入文本
在"输入文本"文本框中输入或粘贴要改写的内容。

**支持：**
- ✅ 任意长度文本（无限制）
- ✅ 中文、英文、多语言
- ✅ 换行和格式保留

#### 2. 选择改写模式
从"改写模式"下拉框中选择合适的模式：

| 模式 | 说明 | 适用场景 |
|------|------|---------|
| 标准改写 | 保持原意，改变表达 | 通用改写 |
| 仿人类写作 ⭐ | 降低 AI 检测风险 | 需要通过 AI 检测 |
| 混合改写 ⭐ | 多种技巧结合 | 高质量改写 |
| 深度重写 ⭐ | 完全重构内容 | 深度改写 |
| 创意改写 | 更生动有趣的表达 | 营销文案 |
| 正式改写 | 商务/学术风格 | 正式场合 |
| 简化改写 | 通俗易懂 | 科普文章 |
| 翻译改写 | 通过翻译达到改写效果 | 快速改写 |

**提示：** 带 ⭐ 标记的模式具有反 AI 检测能力。

#### 3. 选择 AI 模型（可选）
从"AI 模型"下拉框中选择模型：

| 模型 | 特点 | 推荐场景 |
|------|------|---------|
| Claude Haiku 4.5 | 最快速，成本最低 | 简单改写 |
| Claude Sonnet 4.5 ⭐ | 平衡性能和成本 | 日常使用（推荐） |
| Claude Sonnet 4.5 Thinking | 深入分析 | 复杂内容 |
| Claude Opus 4.5 | 强大性能 | 高质量要求 |

**提示：** 如果不选择，系统会使用你在"系统配置"中设置的默认模型。

#### 4. 开始改写
点击"开始改写"按钮，等待改写完成。

**改写过程：**
- 显示"正在改写中，请稍候..."加载动画
- 通常需要 2-10 秒（取决于文本长度和模型）
- 改写完成后自动显示结果

#### 5. 查看结果
改写完成后，会显示：

- **改写后的文本** - 可以直接复制使用
- **统计信息**：
  - 耗时（秒）
  - 相似度（百分比，越低表示改写程度越大）
  - 输入 Tokens（消耗的输入令牌数）
  - 输出 Tokens（消耗的输出令牌数）

#### 6. 复制结果
点击"复制结果"按钮，将改写后的文本复制到剪贴板。

## 🔌 方式二：API 接口调用

### 接口信息

**API 地址：** `http://your-server:8000/api/rewrite`  
**请求方式：** `POST`  
**返回格式：** `JSON`（兼容 5118 格式）

### 认证方式

每次调用需要提供 API Key 进行认证，支持三种方式：

#### 方式 1：HTTP Header（推荐）
```bash
curl -X POST http://localhost:8000/api/rewrite \
  -H "Authorization: sk_your_api_key_here" \
  -H "Content-Type: application/json" \
  -d '{"txt":"要改写的文本","mode":"humanlike","sim":1}'
```

#### 方式 2：请求体参数
```bash
curl -X POST http://localhost:8000/api/rewrite \
  -H "Content-Type: application/json" \
  -d '{"txt":"要改写的文本","mode":"humanlike","api_key":"sk_your_api_key_here","sim":1}'
```

#### 方式 3：URL 参数
```bash
curl -X POST "http://localhost:8000/api/rewrite?api_key=sk_your_api_key_here" \
  -H "Content-Type: application/json" \
  -d '{"txt":"要改写的文本","mode":"humanlike","sim":1}'
```

### 获取 API Key

1. 登录 Web 界面
2. 点击左侧菜单"系统配置"
3. 在"外部API配置"子标签页中，查看"你的 API Key"
4. 点击"复制"按钮复制 API Key

**API Key 格式：** `sk_` 开头的 64 位十六进制字符串

### 请求参数

#### JSON 格式（推荐）

```json
{
  "txt": "要改写的文本内容",
  "mode": "humanlike",
  "model": "claude-sonnet-4-5-20250929",
  "sim": 1
}
```

#### 表单格式（小旋风兼容）

```
txt=要改写的文本内容&mode=humanlike&sim=1
```

#### 参数说明

| 参数 | 类型 | 必填 | 说明 | 示例 |
|------|------|------|------|------|
| `txt` / `content` | String | ✅ 是 | 要改写的文本内容 | "这是要改写的文本" |
| `mode` | String | ❌ 否 | 改写模式 | "humanlike" |
| `model` | String | ❌ 否 | AI 模型 | "claude-sonnet-4-5-20250929" |
| `sim` | Integer | ❌ 否 | 是否返回相似度（1=是，0=否） | 1 |
| `compact` | Integer | ❌ 否 | 是否压缩输出（1=是，0=否） | 0 |

**注意：**
- 如果不指定 `mode` 和 `model`，系统会使用用户在"默认调用配置"中设置的默认值
- `compact=1` 用于压缩输出，适用于小旋风长文本场景

### 返回格式

#### 成功返回

```json
{
  "errcode": "0",
  "errmsg": "",
  "data": "改写后的文本内容",
  "like": "0.5521",
  "_meta": {
    "mode": "humanlike",
    "model": "claude-sonnet-4-5-20250929",
    "duration": 2.5,
    "usage": {
      "input_tokens": 100,
      "output_tokens": 150
    }
  }
}
```

#### 失败返回

```json
{
  "errcode": "100203",
  "errmsg": "API Key 无效",
  "data": ""
}
```

#### 返回字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `errcode` | String | 错误码（"0" 表示成功） |
| `errmsg` | String | 错误信息（成功时为空） |
| `data` | String | 改写后的文本内容 |
| `like` | String | 相似度（0.0000-1.0000，仅当 sim=1 时返回） |
| `_meta` | Object | 元数据（可选） |
| `_meta.mode` | String | 使用的改写模式 |
| `_meta.model` | String | 使用的 AI 模型 |
| `_meta.duration` | Number | 耗时（秒） |
| `_meta.usage` | Object | Token 使用情况 |

### 错误码说明

| 错误码 | 说明 | 解决方法 |
|--------|------|---------|
| 0 | 成功 | - |
| 100202 | 请求缺少 apikey | 添加 API Key |
| 100203 | API Key 无效 | 检查 API Key 是否正确 |
| 200201 | 传进参数为空 | 检查 txt 参数 |
| 100301 | 改写失败 | 检查网络和 API 配置 |

### 编程语言示例

#### Python

```python
import requests

url = "http://localhost:8000/api/rewrite"
headers = {
    "Authorization": "sk_your_api_key_here",
    "Content-Type": "application/json"
}
data = {
    "txt": "这是要改写的文本内容",
    "mode": "humanlike",
    "sim": 1
}

response = requests.post(url, headers=headers, json=data)
result = response.json()

if result["errcode"] == "0":
    print("改写成功：")
    print(result["data"])
    print(f"相似度：{float(result['like']) * 100:.1f}%")
else:
    print(f"改写失败：{result['errmsg']}")
```

#### JavaScript (Node.js)

```javascript
const axios = require('axios');

const url = 'http://localhost:8000/api/rewrite';
const headers = {
    'Authorization': 'sk_your_api_key_here',
    'Content-Type': 'application/json'
};
const data = {
    txt: '这是要改写的文本内容',
    mode: 'humanlike',
    sim: 1
};

axios.post(url, data, { headers })
    .then(response => {
        const result = response.data;
        if (result.errcode === '0') {
            console.log('改写成功：');
            console.log(result.data);
            console.log(`相似度：${(parseFloat(result.like) * 100).toFixed(1)}%`);
        } else {
            console.log(`改写失败：${result.errmsg}`);
        }
    })
    .catch(error => {
        console.error('请求失败：', error.message);
    });
```

#### PHP

```php
<?php
$url = 'http://localhost:8000/api/rewrite';
$headers = [
    'Authorization: sk_your_api_key_here',
    'Content-Type: application/json'
];
$data = json_encode([
    'txt' => '这是要改写的文本内容',
    'mode' => 'humanlike',
    'sim' => 1
]);

$ch = curl_init($url);
curl_setopt($ch, CURLOPT_POST, true);
curl_setopt($ch, CURLOPT_POSTFIELDS, $data);
curl_setopt($ch, CURLOPT_HTTPHEADER, $headers);
curl_setopt($ch, CURLOPT_RETURNTRANSFER, true);

$response = curl_exec($ch);
curl_close($ch);

$result = json_decode($response, true);

if ($result['errcode'] === '0') {
    echo "改写成功：\n";
    echo $result['data'] . "\n";
    echo "相似度：" . (floatval($result['like']) * 100) . "%\n";
} else {
    echo "改写失败：" . $result['errmsg'] . "\n";
}
?>
```

## ⚙️ 系统配置

### 配置默认 API 和模型

为了简化使用，你可以在系统中配置默认的 API 和模型：

1. 登录 Web 界面
2. 点击左侧菜单"系统配置"
3. 切换到"默认调用配置"子标签页
4. 配置以下内容：
   - **默认 API 配置** - 选择要使用的 API（需要先在"外部API配置"中添加）
   - **默认模型** - 选择默认使用的 AI 模型
   - **默认改写方式** - 选择默认的改写模式
5. 点击"保存默认配置"

**好处：**
- API 调用时如果不指定 `mode` 和 `model`，会自动使用默认配置
- Web 界面改写时会自动选中默认模式和模型
- 简化配置，提高效率

### 配置外部 API

如果你想使用自己的 AI API（如 OpenAI、Claude、Gemini 等）：

1. 登录 Web 界面
2. 点击左侧菜单"系统配置"
3. 在"外部API配置"子标签页中点击"添加新配置"
4. 填写配置信息：
   - **配置名称** - 自定义名称
   - **API 类型** - 选择 API 类型（OpenAI、Claude、Gemini 等）
   - **Base URL** - API 地址
   - **API Key** - API 密钥
5. 点击"测试连接"验证配置
6. 点击"保存"

**支持的 API 类型：**
- OpenAI / OpenAI 兼容（国内中转）
- Anthropic Claude 官方
- Google Gemini
- Cohere
- 阿里云通义千问
- 百度文心一言
- Azure OpenAI

详细配置请参考：[AI服务快速配置.md](./AI服务快速配置.md)

## 🎯 自定义改写模式

除了内置的改写模式，你还可以自定义自己的改写模式：

### 通过 Web 界面添加

1. 登录 Web 界面
2. 点击左侧菜单"改写模式"
3. 点击"+ 添加新模式"
4. 填写模式信息：
   - **模式 ID** - 英文标识符（如 `my_mode`）
   - **模式名称** - 显示名称（如 "我的自定义模式"）
   - **模式描述** - 简短描述
   - **改写指令** - 告诉 AI 如何改写（最重要）
   - **反 AI 检测** - 是否具有反 AI 检测能力
5. 点击"保存"

### 改写指令示例

#### 小红书风格
```
请将以下文本改写成小红书风格：
1. 使用emoji表情
2. 分段清晰，每段不超过3行
3. 语气活泼可爱
4. 使用网络用语
5. 结尾加互动问题
```

#### 新闻报道风格
```
请将以下文本改写成新闻报道风格：
1. 客观中立的语气
2. 使用新闻常用词汇
3. 结构清晰：导语-主体-结尾
4. 避免主观评价
5. 使用准确的数据和事实
```

#### SEO 优化风格
```
请将以下文本改写成 SEO 优化风格：
1. 保留关键词
2. 增加相关长尾词
3. 段落清晰，便于阅读
4. 使用小标题
5. 适当增加内链锚文本
```

## ❓ 常见问题

### Q1: Web 界面改写失败，提示"用户未配置 API Key"？

**A:** 你需要先配置外部 API：
1. 进入"系统配置" → "外部API配置"
2. 添加一个 API 配置（如 Claude、OpenAI 等）
3. 在"默认调用配置"中选择该 API 作为默认

### Q2: API 调用失败，提示"API Key 无效"？

**A:** 检查以下几点：
1. 确认 API Key 格式正确（`sk_` 开头）
2. 确认 API Key 没有过期
3. 在 Web 界面"系统配置"中查看你的 API Key
4. 确认 Authorization Header 格式正确

### Q3: 小旋风调用失败？

**A:** 请参考：
- [小旋风配置教程.md](./小旋风配置教程.md)
- [小旋风问题诊断卡.txt](./小旋风问题诊断卡.txt)
- [小旋风长文本问题解决方案.md](./小旋风长文本问题解决方案.md)

### Q4: 改写结果相似度太高？

**A:** 尝试以下方法：
1. 使用"深度重写"或"混合改写"模式
2. 使用更强大的模型（如 Claude Opus）
3. 自定义改写模式，增加改写力度

### Q5: 改写速度太慢？

**A:** 
1. 使用更快的模型（如 Claude Haiku）
2. 减少文本长度
3. 检查网络连接
4. 检查 API 服务状态

### Q6: 如何批量改写文本？

**A:** 
1. 使用 API 接口编写脚本
2. 使用小旋风等工具集成
3. 参考上面的编程语言示例
