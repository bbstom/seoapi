# 如何查看请求日志

## ❓ 为什么看不到日志？

启动服务后只会显示启动信息，**只有当有请求进来时才会显示请求日志**。

## ✅ 正确的测试步骤

### 步骤 1：启动服务

打开 PowerShell 或 CMD：

```powershell
cd seoapi-nodejs
node server.js
```

你会看到：

```
========================================
  SEO API - Claude AI 文本改写服务
========================================

API 文档: http://localhost:8000/api/docs
Web 界面: http://localhost:8000

按 Ctrl+C 停止服务
========================================

Trust proxy: loopback (开发环境)
API Key 状态: ✓ 已配置
```

**此时还没有请求日志，这是正常的！**

### 步骤 2：发送测试请求

**方法 1：使用测试脚本（推荐）**

打开**另一个** PowerShell 窗口：

```powershell
cd seoapi-nodejs
.\test-api.ps1
```

或双击运行 `test-api.bat`

**方法 2：使用 curl**

```powershell
curl http://localhost:8000/health
```

**方法 3：在浏览器中访问**

打开浏览器访问：`http://localhost:8000/health`

**方法 4：在小旋风中测试**

在小旋风中点击"测试"按钮

## 🎯 完整测试流程

### 使用两个窗口

**窗口 1：运行服务**

```powershell
cd seoapi-nodejs
node server.js
```

保持这个窗口打开，用于查看日志。

**窗口 2：发送测试请求**

```powershell
cd seoapi-nodejs
.\test-api.ps1
```

或者：

```powershell
curl -X POST http://localhost:8000/api/rewrite `
  -H "Authorization: sk_f7747efeefece9fbeb79bfecd825e01ff427c2f343acd398f6f3713469906d00" `
  -H "Content-Type: application/x-www-form-urlencoded" `
  -d "txt=测试文本&mode=seo_fast&sim=1"
```

### 使用 VS Code 分屏

1. 在 VS Code 中打开项目
2. 按 `` Ctrl + ` `` 打开终端
3. 点击终端右上角的 `+` 按钮创建新终端
4. 在第一个终端运行：`node server.js`
5. 在第二个终端运行：`.\test-api.ps1`
6. 在第一个终端查看日志

## ⚠️ 常见问题

### Q1: 启动后看不到日志？

**A:** 这是正常的！只有当有请求进来时才会显示请求日志。

**解决：** 发送一个测试请求（使用 curl、浏览器或小旋风）

### Q2: 发送请求后还是看不到日志？

**A:** 可能的原因：

1. **请求没有到达服务器**
   - 检查服务是否正在运行
   - 检查端口是否正确（8000）
   - 检查防火墙设置

2. **请求被拦截**
   - 检查 API Key 是否正确
   - 查看是否有错误提示

3. **日志被重定向**
   - 确保没有使用 `> logs.txt` 重定向
   - 确保在正确的窗口查看

**解决：**

```powershell
# 测试服务是否运行
curl http://localhost:8000/health

# 如果返回 {"status":"ok",...} 说明服务正常
```

### Q3: 只看到启动信息，没有请求日志？

**A:** 需要发送请求才会有请求日志。

**测试方法：**

```powershell
# 方法 1：使用测试脚本
.\test-api.ps1

# 方法 2：使用 curl
curl http://localhost:8000/health

# 方法 3：在浏览器中访问
# 打开 http://localhost:8000/health
```

### Q4: 小旋风测试时看不到日志？

**A:** 检查以下几点：

1. **服务是否在运行**
   ```powershell
   # 应该看到 node server.js 进程
   ```

2. **小旋风配置是否正确**
   - API 地址：`http://localhost:8000/api/rewrite`
   - Authorization：正确的 API Key

3. **查看小旋风的错误提示**
   - 如果提示"连接失败"，检查服务是否运行
   - 如果提示"返回空内容"，查看服务器日志

## 💡 推荐配置

### 开发调试推荐

**使用 VS Code：**

1. 在 VS Code 中打开项目
2. 按 `` Ctrl + ` `` 打开终端
3. 点击 `+` 创建两个终端
4. 终端 1：`node server.js`（查看日志）
5. 终端 2：`.\test-api.ps1`（发送测试）

**使用 Windows Terminal：**

1. 打开 Windows Terminal
2. 按 `Ctrl + Shift + D` 分屏
3. 左边：`node server.js`
4. 右边：`.\test-api.ps1`

## 🎉 总结

**记住：**
1. ✅ 启动服务后只显示启动信息
2. ✅ 发送请求后才会显示请求日志
3. ✅ 使用两个窗口：一个运行服务，一个发送请求
4. ✅ 在运行服务的窗口查看日志

**快速测试：**
```powershell
# 窗口 1
node server.js

# 窗口 2

```
