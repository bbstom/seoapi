# 运维快速参考

## 🚀 常用命令速查

### PM2 进程管理

```bash
# 查看服务状态
pm2 status

# 查看日志（实时）
pm2 logs seoapi

# 查看最近100行日志
pm2 logs seoapi --lines 100

# 查看错误日志
pm2 logs seoapi --err

# 重启服务
pm2 restart seoapi

# 重载服务（零停机）
pm2 reload seoapi

# 停止服务
pm2 stop seoapi

# 启动服务
pm2 start seoapi

# 删除服务
pm2 delete seoapi

# 实时监控
pm2 monit

# 查看详细信息
pm2 show seoapi
```

### 数据库操作

```bash
# 登录数据库
mysql -u seoapi_user -p seoapi

# 备份数据库
mysqldump -u seoapi_user -p seoapi | gzip > backup_$(date +%Y%m%d).sql.gz

# 恢复数据库
gunzip < backup_20260312.sql.gz | mysql -u seoapi_user -p seoapi

# 查看数据库大小
mysql -u seoapi_user -p -e "SELECT table_schema AS 'Database', ROUND(SUM(data_length + index_length) / 1024 / 1024, 2) AS 'Size (MB)' FROM information_schema.TABLES WHERE table_schema = 'seoapi';"

# 优化数据库
mysql -u seoapi_user -p seoapi -e "OPTIMIZE TABLE api_logs, request_stats, node_health_logs, failover_logs;"
```

### Nginx 操作

```bash
# 测试配置
sudo nginx -t

# 重启 Nginx
sudo systemctl restart nginx

# 重载配置（不中断服务）
sudo systemctl reload nginx

# 查看状态
sudo systemctl status nginx

# 查看错误日志
sudo tail -f /var/log/nginx/error.log

# 查看访问日志
sudo tail -f /var/log/nginx/access.log
```

### 系统监控

```bash
# 查看系统资源
htop

# 查看磁盘使用
df -h

# 查看内存使用
free -h

# 查看端口占用
sudo netstat -tlnp | grep 8000

# 查看进程
ps aux | grep node

# 查看系统日志
sudo journalctl -u nginx -f
```

## 🔧 常见问题处理

### 问题1：服务无法启动

```bash
# 1. 查看详细日志
pm2 logs seoapi --lines 200

# 2. 检查端口占用
sudo netstat -tlnp | grep 8000

# 3. 手动启动测试
cd /var/www/seoapi
node server.js

# 4. 检查环境变量
cat .env

# 5. 检查数据库连接
mysql -u seoapi_user -p seoapi
```

### 问题2：数据库连接失败

```bash
# 1. 检查 MySQL 状态
sudo systemctl status mysql

# 2. 重启 MySQL
sudo systemctl restart mysql

# 3. 查看 MySQL 日志
sudo tail -f /var/log/mysql/error.log

# 4. 测试连接
mysql -u seoapi_user -p -h localhost seoapi

# 5. 检查用户权限
mysql -u root -p -e "SHOW GRANTS FOR 'seoapi_user'@'localhost';"
```

### 问题3：Nginx 502 错误

```bash
# 1. 检查 Node.js 服务
pm2 status

# 2. 重启 Node.js 服务
pm2 restart seoapi

# 3. 检查 Nginx 配置
sudo nginx -t

# 4. 查看 Nginx 错误日志
sudo tail -f /var/log/nginx/error.log

# 5. 重启 Nginx
sudo systemctl restart nginx
```

### 问题4：内存占用过高

```bash
# 1. 查看内存使用
pm2 monit

# 2. 重启服务释放内存
pm2 restart seoapi

# 3. 设置内存限制
pm2 delete seoapi
pm2 start server.js --name seoapi --max-memory-restart 1G

# 4. 清理日志
pm2 flush seoapi
```

### 问题5：磁盘空间不足

```bash
# 1. 查看磁盘使用
df -h

# 2. 查找大文件
du -sh /var/www/seoapi/* | sort -rh | head -10

# 3. 清理日志
cd /var/www/seoapi/logs
rm -f *.log.old

# 4. 清理旧备份
find /var/backups/seoapi -name "*.sql.gz" -mtime +30 -delete

# 5. 清理 PM2 日志
pm2 flush
```

## 📊 性能优化

### 数据库优化

```sql
-- 查看慢查询
SHOW VARIABLES LIKE 'slow_query_log';

-- 查看表大小
SELECT 
    table_name AS 'Table',
    ROUND(((data_length + index_length) / 1024 / 1024), 2) AS 'Size (MB)'
FROM information_schema.TABLES
WHERE table_schema = 'seoapi'
ORDER BY (data_length + index_length) DESC;

-- 优化表
OPTIMIZE TABLE api_logs;
OPTIMIZE TABLE request_stats;

-- 分析表
ANALYZE TABLE api_logs;

-- 查看索引使用情况
SHOW INDEX FROM api_logs;
```

### 清理旧数据

```sql
-- 删除30天前的日志（谨慎操作！）
DELETE FROM api_logs WHERE created_at < DATE_SUB(NOW(), INTERVAL 30 DAY);

-- 删除旧的健康检查日志
DELETE FROM node_health_logs WHERE checked_at < DATE_SUB(NOW(), INTERVAL 7 DAY);

-- 删除旧的故障转移日志
DELETE FROM failover_logs WHERE failover_time < DATE_SUB(NOW(), INTERVAL 30 DAY);
```


## 📦 备份和恢复

### 完整备份

```bash
# 1. 备份数据库
mysqldump -u seoapi_user -p seoapi | gzip > /var/backups/seoapi/db_$(date +%Y%m%d).sql.gz

# 2. 备份配置文件
tar -czf /var/backups/seoapi/config_$(date +%Y%m%d).tar.gz /var/www/seoapi/.env /var/www/seoapi/data/

# 3. 备份日志（可选）
tar -czf /var/backups/seoapi/logs_$(date +%Y%m%d).tar.gz /var/www/seoapi/logs/
```

### 快速恢复

```bash
# 1. 停止服务
pm2 stop seoapi

# 2. 恢复数据库
gunzip < /var/backups/seoapi/db_20260312.sql.gz | mysql -u seoapi_user -p seoapi

# 3. 恢复配置
tar -xzf /var/backups/seoapi/config_20260312.tar.gz -C /

# 4. 启动服务
pm2 start seoapi

# 5. 检查状态
pm2 status
pm2 logs seoapi --lines 50
```

## 🔄 更新部署

### 更新代码

```bash
# 1. 进入项目目录
cd /var/www/seoapi

# 2. 备份当前版本
tar -czf /var/backups/seoapi/code_$(date +%Y%m%d).tar.gz .

# 3. 拉取最新代码
git pull origin main

# 4. 安装新依赖
npm install --production

# 5. 重启服务
pm2 restart seoapi

# 6. 查看日志
pm2 logs seoapi --lines 50
```

### 回滚版本

```bash
# 1. 停止服务
pm2 stop seoapi

# 2. 恢复代码
cd /var/www
rm -rf seoapi
tar -xzf /var/backups/seoapi/code_20260312.tar.gz -C seoapi/

# 3. 启动服务
pm2 start seoapi

# 4. 检查状态
pm2 status
```

## 📞 紧急联系

### 服务完全宕机

```bash
# 1. 检查服务器状态
ping your-server-ip

# 2. SSH 登录服务器
ssh user@your-server-ip

# 3. 检查所有服务
sudo systemctl status mysql
sudo systemctl status nginx
pm2 status

# 4. 查看系统日志
sudo journalctl -xe

# 5. 重启所有服务
sudo systemctl restart mysql
sudo systemctl restart nginx
pm2 restart all
```

### 数据库损坏

```bash
# 1. 停止服务
pm2 stop seoapi

# 2. 检查数据库
mysqlcheck -u seoapi_user -p --auto-repair seoapi

# 3. 如果无法修复，恢复备份
gunzip < /var/backups/seoapi/db_latest.sql.gz | mysql -u seoapi_user -p seoapi

# 4. 启动服务
pm2 start seoapi
```
