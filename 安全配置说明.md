# 安全配置说明

## 🔒 已实现的安全措施

### 1. 密码安全

**bcrypt 加密**
- 使用 bcrypt 替代 SHA256
- 自动加盐（salt rounds: 10）
- 防止彩虹表攻击
- 防止暴力破解

```javascript
// 密码哈希
const hashedPassword = await bcrypt.hash(password, 10);

// 密码验证
const isValid = await bcrypt.compare(password, hashedPassword);
```

### 2. 速率限制

**登录限制**
- 15分钟内最多5次登录尝试
- 超过限制后需等待15分钟
- 防止暴力破解

**API 调用限制**
- 1分钟内最多60次请求
- 防止滥用和 DDoS 攻击

### 3. 会话管理

**安全特性**
- 会话 Token 随机生成（32字节）
- 7天自动过期
- 登出时立即失效
- 每次请求验证会话有效性

### 4. API Key 认证

**多层验证**
- 每个用户独立的 API Key
- 支持 Header、Body、Query 三种认证方式
- API Key 格式：`sk_` + 64位随机字符

### 5. 权限控制

**角色分离**
- 管理员（admin）：可以管理所有用户
- 普通用户（user）：只能管理自己的配置

### 6. 数据验证

**输入验证**
- 用户名和密码非空检查
- 密码长度最少6位
- URL 格式验证
- 文本长度限制（<5000字符）

## ⚠️ 需要注意的安全问题

### 1. 敏感数据存储

**当前状态**：Claude API Key 明文存储在 JSON 文件中

**风险**：如果服务器被入侵，API Key 可能泄露

**建议**：
- 设置严格的文件权限（chmod 600）
- 定期更换 API Key
- 考虑使用环境变量

### 2. HTTPS 传输

**当前状态**：默认使用 HTTP

**风险**：数据传输可能被窃听

**建议**：生产环境必须使用 HTTPS

### 3. 文件权限

**重要**：确保 data 目录只有服务进程可以访问

```bash
# Linux 设置
chmod 700 data
chmod 600 data/*.json
chown node_user:node_user data -R
```

## 🛡️ 安全最佳实践

### 1. 修改默认密码

**首次登录后立即修改**
```
默认账号：admin / admin123
```

### 2. 定期更换密钥

**建议频率**
- 密码：每3个月
- API Key：每月或泄露时
- Claude API Key：每月

### 3. 监控异常登录

**关注日志**
```bash
# 查看登录记录
grep "登录" logs/app.log

# 查看失败的登录尝试
grep "登录失败" logs/app.log
```

### 4. 备份数据

**定期备份**
```bash
# 每天备份
cp -r data data_backup_$(date +%Y%m%d)

# 保留最近7天的备份
find data_backup_* -mtime +7 -delete
```

### 5. 限制访问

**防火墙配置**
```bash
# 只允许特定 IP 访问
iptables -A INPUT -p tcp --dport 8000 -s 192.168.1.0/24 -j ACCEPT
iptables -A INPUT -p tcp --dport 8000 -j DROP
```

### 6. 使用 HTTPS

**Nginx 配置示例**
```nginx
server {
    listen 443 ssl;
    server_name your-domain.com;
    
    ssl_certificate /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;
    
    location / {
        proxy_pass http://localhost:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

## 🚨 安全事件响应

### 如果发现异常登录

1. **立即行动**
   ```bash
   # 停止服务
   Stop-Process -Name node -Force
   
   # 检查日志
   cat logs/app.log | grep "登录"
   
   # 重置所有密码
   rm data/users.json
   rm data/sessions.json
   ```

2. **重新配置**
   - 重启服务（会创建新的默认账号）
   - 修改默认密码
   - 重新生成所有 API Key

### 如果 API Key 泄露

1. **立即重新生成**
   - 登录 Web 界面
   - 进入配置页面
   - 点击"重新生成 API Key"

2. **更新小旋风配置**
   - 使用新的 API Key

### 如果 Claude API Key 泄露

1. **在 Anthropic Console 中撤销旧 Key**
2. **生成新的 API Key**
3. **在系统中更新配置**
