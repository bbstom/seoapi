## 2026-03-08 - 优化 Trust Proxy 配置

### 🔧 问题修复

#### 修复开发环境 Trust Proxy 安全警告

**问题描述：**
```
ValidationError: The Express 'trust proxy' setting is true, 
which allows anyone to trivially bypass IP-based rate limiting.
```

**问题原因：**
- 使用 `trust proxy = true` 在所有环境
- 开发环境（本地测试）会触发安全警告
- 生产环境安全性不够（信任所有代理）

**解决方案：**

根据环境自动配置 trust proxy：

```javascript
// 配置 trust proxy（根据环境自动调整）
if (process.env.NODE_ENV === 'production') {
    // 生产环境：只信任 1 层代理（更安全）
    app.set('trust proxy', 1);
} else {
    // 开发环境：信任本地代理
    app.set('trust proxy', 'loopback');
}
```

**环境配置：**

在 `.env` 文件中设置：

```env
# 开发环境（本地测试）
NODE_ENV=development

# 生产环境（服务器部署）
NODE_ENV=production
```

**影响：**
- ✅ 开发环境不再触发安全警告
- ✅ 生产环境更安全（只信任 1 层代理）
- ✅ 自动适配不同环境
- ✅ 速率限制正常工作

**配置对比：**

| 环境 | NODE_ENV | Trust Proxy | 安全警告 |
|------|----------|-------------|---------|
| 开发 | development | loopback | 无 |
| 生产 | production | 1 | 无 |

## 2026-03-08 - 修复小旋风改写失败问题

### 🔧 问题修复

#### 修复小旋风"改写失败"但后台显示成功的问题

**问题描述：**
- 后台日志显示改写成功
- 小旋风提示"改写失败"
- 部分请求成功，部分请求失败

**问题分析：**

1. **响应格式问题**
   - 响应包含 `_meta` 字段
   - 小旋风可能不支持额外字段
   - 导致解析失败

2. **响应内容问题**
   - 可能包含 BOM 字符（\uFEFF）
   - 可能包含不可见的控制字符
   - 导致小旋风解析错误

3. **响应体大小**
   - 包含 `_meta` 增加了响应体积
   - 长文本 + 元数据可能超过小旋风限制

**解决方案：**

1. **简化响应格式**
```javascript
// 默认返回纯净的 5118 格式
{
  "errcode": "0",
  "errmsg": "",
  "data": "改写后的文本",
  "like": "0.5521"
}

// 只在开发环境或 debug=1 时返回 _meta
```

2. **清理响应文本**
```javascript
// 移除 BOM 和不可见字符
rewrittenText = rewrittenText.replace(/^\uFEFF/, '').trim();
```

3. **优化兼容性**
   - 移除了默认的 `_meta` 字段
   - 减小响应体积
   - 提高小旋风兼容性

**影响：**
- ✅ 修复了小旋风改写失败问题
- ✅ 响应格式更符合 5118 标准
- ✅ 响应体积减小约 30%

## 2026-03-08 - 修复反向代理配置问题

### 🔧 问题修复

#### 修复 X-Forwarded-For 错误

**问题描述：**
```
ValidationError: The 'X-Forwarded-For' header is set but the Express 'trust proxy' setting is false (default).
```

**问题原因：**
- 服务部署在 Nginx 反向代理后面
- Nginx 添加了 `X-Forwarded-For` 头传递真实客户端 IP
- Express 默认不信任代理（`trust proxy = false`）
- `express-rate-limit` 无法正确识别用户 IP
- 导致速率限制失效和错误

**解决方案：**

在 `server.js` 中添加：
```javascript
// 信任反向代理（Nginx、Apache 等）
app.set('trust proxy', true);
```

**影响：**
- ✅ 修复了 ValidationError 错误
- ✅ 速率限制现在可以正确识别用户
- ✅ 日志显示真实客户端 IP

## 2026-03-08 - 超时问题解决方案

### 📚 新增文档

#### 504 Gateway Timeout 超时问题完整解决方案

**问题描述：**
- 使用 `seo_fast` 模式 + `claude-opus-4-6` 模型
- 文本长度 4812 字符
- 错误：`504 Gateway Time-out` (nginx/1.26.3)

**根本原因：**
- Opus 4.6 是最慢的模型
- 处理 4812 字需要 70-90 秒
- api123.icu nginx 超时限制 60 秒
- 导致超时错误

**解决方案：**

1. **更换模型（推荐）⭐**
   - 使用 Sonnet 4.5 或 Haiku 4.5
   - Sonnet 4.5 处理 4812 字约 30-40 秒 ✅
   - Haiku 4.5 处理 4812 字约 15-20 秒 ✅

2. **按文本长度选择模型**
   - < 500 字：可用 Opus 4.6
   - 500-2000 字：推荐 Sonnet 4.5
   - 2000-5000 字：推荐 Sonnet 4.5 + seo_fast
   - > 5000 字：推荐 Haiku 4.5

3. **设置默认配置**
   - Web 界面设置默认模型为 Sonnet 4.5
   - 避免意外使用 Opus 4.6 处理长文本

4. **分段处理**
   - 超长文本（> 5000 字）分段改写
   - 每段 2000-3000 字最佳

**模型速度对比：**

| 模型 | 速度 | 适用文本长度 | 超时风险 |
|------|------|--------------|----------|
| Haiku 4.5 | ⚡⚡⚡⚡⚡ | 0-10000 字 | 🟢 无风险 |
| Sonnet 4.5 | ⚡⚡⚡⚡ | 0-5000 字 | 🟢 低风险 |
| Sonnet 4.6 | ⚡⚡⚡ | 0-3000 字 | 🟡 中风险 |
| Opus 4.5 | ⚡⚡ | 0-2000 字 | 🟡 中风险 |
| Opus 4.6 | ⚡ | 0-1000 字 | 🔴 高风险 |

**推荐配置：**

```
默认模型: claude-sonnet-4-5-20250929
默认模式: seo_spider
小旋风 post 格式: txt={word}&mode=seo_spider&sim=1
```

**关键要点：**
- ⚠️ Opus 4.6 仅适合短文本（< 1000 字）
- ✅ Sonnet 4.5 适合大部分场景（< 5000 字）
- ✅ Haiku 4.5 适合长文本和批量处理
- ✅ 根据文本长度选择合适的模型

**原因：**
- 提示词中包含敏感词汇（"伪原创"、"避开查重"、"规避系统"）
- 触发了 Claude 的安全机制

**解决方案：**
- 移除所有敏感词汇
- 改用正面、建设性的表述

## 2026-03-08 - 默认配置功能

### 🎉 新增功能

#### Web 后台默认配置

现在可以在 Web 后台为每个用户设置默认的模型和改写模式！

**功能亮点：**

1. **简化小旋风配置**
   - 之前：`txt={word}&mode=humanlike&model=claude-sonnet-4-5-20250929&sim=1`
   - 现在：`txt={word}&sim=1`（使用默认配置）

2. **灵活的参数覆盖**
   - 可以随时在请求中指定参数覆盖默认值
   - 支持部分覆盖（只指定模式或只指定模型）

3. **多用户独立配置**
   - 每个用户可以设置自己的默认模型和模式
   - 适合团队协作场景

4. **实时生效**
   - 修改默认配置后立即生效
   - 无需重启服务

### 📝 配置方式

1. 登录 Web 后台（http://localhost:8000）
2. 进入"配置"标签页
3. 在"默认调用配置"区域设置：
   - 默认模型（推荐：Claude Sonnet 4.5）
   - 默认改写模式（推荐：humanlike）
4. 点击"保存默认配置"

### 🔄 配置优先级

```
指定的参数 > 用户默认配置 > 系统默认值
```

**示例：**

| 请求参数 | 使用的模型 | 使用的模式 |
|---------|-----------|-----------|
| `txt={word}&sim=1` | 用户默认 | 用户默认 |
| `txt={word}&mode=creative&sim=1` | 用户默认 | creative |
| `txt={word}&model=claude-haiku-4-5-20251001&sim=1` | claude-haiku-4-5-20251001 | 用户默认 |
| `txt={word}&mode=formal&model=claude-opus-4-6&sim=1` | claude-opus-4-6 | formal |

- **优点**：响应速度最快，成本最低
