# 认证系统说明

## 🔐 系统特性

- ✅ 用户登录认证
- ✅ 每个用户独立的 API Key
- ✅ 每个用户独立的 Claude API Key
- ✅ 管理员可以管理所有用户
- ✅ 支持小旋风 Authorization 认证
- ✅ 会话管理（7天有效期）

## 👤 默认管理员账号

首次启动时会自动创建管理员账号：

```
用户名: admin
密码: admin123
```

**重要：** 首次登录后请立即修改密码！

## 🔑 API Key 认证方式

### 方式 1: Header 认证（推荐）

```bash
curl -X POST http://your-server:8000/api/rewrite \
  -H "Authorization: sk_your_api_key_here" \
  -H "Content-Type: application/json" \
  -d '{"txt":"测试文本","mode":"humanlike"}'
```

### 方式 2: 请求体认证

```bash
curl -X POST http://your-server:8000/api/rewrite \
  -H "Content-Type: application/json" \
  -d '{"txt":"测试文本","mode":"humanlike","api_key":"sk_your_api_key_here"}'
```

### 方式 3: URL 参数认证

```bash
curl -X POST "http://your-server:8000/api/rewrite?api_key=sk_your_api_key_here" \
  -H "Content-Type: application/json" \
  -d '{"txt":"测试文本","mode":"humanlike"}'
```

## 🎯 小旋风配置

### 配置示例

```
API名称: Claude AI 伪原创
API地址: http://your-server:8000/api/rewrite
post格式: txt={word}&mode=humanlike&sim=1
Authorization: sk_your_api_key_here
成功标志: errcode=0
返回错误的字段: errmsg
伪原创后的内容字段: data
```

**重要：** 在 Authorization 字段填入你的 API Key

## 📱 Web 界面使用

### 1. 登录

访问：`http://your-server:8000`

输入用户名和密码登录

### 2. 查看 API Key

登录后，在"配置"标签页可以看到你的 API Key

### 3. 配置 Claude API Key

在"配置"标签页输入你的 Claude API Key

### 4. 使用改写功能

在"文本改写"标签页直接使用

## 👥 用户管理（管理员）

### 创建新用户

1. 以管理员身份登录
2. 进入"用户管理"标签页
3. 点击"添加用户"
4. 填写用户名、密码、Claude API Key（可选）
5. 系统自动生成该用户的 API Key

### 查看用户列表

管理员可以看到所有用户及其 API Key

### 删除用户

管理员可以删除普通用户（不能删除 admin）

## 🔄 API Key 管理

### 查看自己的 API Key

登录后在"配置"页面查看

### 重新生成 API Key

如果 API Key 泄露，可以重新生成：

1. 登录 Web 界面
2. 进入"配置"标签页
3. 点击"重新生成 API Key"
4. 旧的 API Key 立即失效

## 🔒 安全建议

### 1. 修改默认密码

```
首次登录后立即修改 admin 密码
```

### 2. 定期更换 API Key

```
建议每月更换一次 API Key
```

### 3. 不要共享 API Key

```
每个用户使用自己的 API Key
不要在公开场合泄露 API Key
```

### 4. 使用 HTTPS

```
生产环境建议配置 HTTPS
防止 API Key 在传输中被窃取
```

### 5. 限制 IP 访问

```
可以在防火墙层面限制访问 IP
只允许信任的 IP 访问
```

## 🔧 API 接口文档

### 登录

```http
POST /api/auth/login
Content-Type: application/json

{
  "username": "admin",
  "password": "admin123"
}

响应：
{
  "success": true,
  "token": "session_token_here",
  "user": {
    "username": "admin",
    "role": "admin",
    "apiKey": "sk_xxx"
  }
}
```

### 获取当前用户信息

```http
GET /api/auth/me
X-Session-Token: your_session_token

响应：
{
  "success": true,
  "user": {
    "username": "admin",
    "role": "admin",
    "apiKey": "sk_xxx",
    "claudeApiKey": "已配置"
  }
}
```

### 创建用户（管理员）

```http
POST /api/users
X-Session-Token: admin_session_token
Content-Type: application/json

{
  "username": "user1",
  "password": "password123",
  "claudeApiKey": "sk-ant-xxx"  // 可选
}

响应：
{
  "success": true,
  "message": "用户创建成功",
  "user": {
    "username": "user1",
    "apiKey": "sk_generated_key"
  }
}
```

### 更新 Claude API Key

```http
POST /api/users/claude-key
X-Session-Token: your_session_token
Content-Type: application/json

{
  "claudeApiKey": "sk-ant-your-key"
}

响应：
{
  "success": true,
  "message": "Claude API Key 更新成功"
}
```

### 重新生成 API Key

```http
POST /api/users/regenerate-key
X-Session-Token: your_session_token

响应：
{
  "success": true,
  "apiKey": "sk_new_generated_key",
  "message": "API Key 已重新生成"
}
```

## 🐛 错误码说明

| 错误码 | 说明 | 解决方法 |
|--------|------|---------|
| 100202 | 请求缺少 apikey | 在 Authorization 头或请求中添加 API Key |
| 100203 | 无效的 apikey | 检查 API Key 是否正确 |
| 100203 | 用户未配置 Claude API Key | 在 Web 界面配置 Claude API Key |
| 401 | 未登录 | 重新登录获取 session token |
| 403 | 权限不足 | 该操作需要管理员权限 |

## 💡 常见问题

### Q1: 忘记密码怎么办？

**A:** 删除 `data/users.json` 文件，重启服务会重新创建默认管理员账号

### Q2: API Key 泄露了怎么办？

**A:** 登录 Web 界面，点击"重新生成 API Key"

### Q3: 可以禁用认证吗？

**A:** 不建议。如果确实需要，可以修改代码移除 `verifyApiKey` 中间件

### Q4: 多个用户可以共用一个 Claude API Key 吗？

**A:** 可以，但不建议。建议每个用户使用自己的 Claude API Key，方便费用管理

### Q5: 会话过期时间可以修改吗？

**A:** 可以，在 `server.js` 中修改：
```javascript
expiresAt: new Date(Date.now() + 7 * 24 * 60 * 60 * 1000) // 7天
```
