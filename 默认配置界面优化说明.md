# 默认配置界面优化说明

## 🎯 优化目标

让 Web 后台的"默认调用配置"区域能够显示所有可用的改写模式，包括新增的 5 个 SEO 专用模式。

## 🐛 问题描述

用户反馈在 Web 后台的"配置"页面看不到完整的默认模式选项，特别是新增的 SEO 专用模式。

**原因**：
- 默认模式下拉框使用的是硬编码的选项列表
- 只包含了 8 个基础模式
- 没有包含新增的 5 个 SEO 专用模式

## ✅ 解决方案

### 1. 动态加载模式列表

将硬编码的选项改为动态加载：

**修改前**：
```html
<select id="default-mode">
    <option value="standard">标准改写</option>
    <option value="humanlike">仿人类写作</option>
    <!-- 只有 8 个固定选项 -->
</select>
```

**修改后**：
```html
<select id="default-mode">
    <!-- 模式列表将动态加载 -->
</select>
```

### 2. 添加加载函数

新增 `loadDefaultModeOptions()` 函数：

```javascript
async function loadDefaultModeOptions() {
    try {
        const response = await fetch('/api/modes');
        const data = await response.json();
        
        const defaultModeElement = document.getElementById('default-mode');
        if (defaultModeElement && data.modes) {
            defaultModeElement.innerHTML = data.modes.map(mode => {
                let badge = '';
                if (mode.antiAI) badge += ' ⭐';
                if (mode.seo) badge += ' 🎯';
                return `<option value="${mode.id}">${mode.name} - ${mode.description}${badge}</option>`;
            }).join('');
        }
    } catch (error) {
        console.error('加载模式列表失败:', error);
    }
}
```

### 3. 在 loadConfig 中调用

```javascript
async function loadConfig() {
    // 先加载模式列表
    await loadDefaultModeOptions();
    
    // 然后加载用户配置
    const response = await authFetch('/api/auth/me');
    // ...
}
```

### 1. 查看所有模式

1. 访问 http://localhost:8000
2. 登录后点击"配置"标签页
3. 滚动到"默认调用配置"区域
4. 点击"默认改写模式"下拉框

### 2. 选择默认模式

**推荐配置**：

**日常养站**：
- 默认模式：seo_spider（SEO 蜘蛛优化）⭐🎯

data.modes.map(mode => {
